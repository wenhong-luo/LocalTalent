LocalTalent 一期演示使用说明

给第一次接手项目的新同事看的本地演示指南：按步骤拉起依赖、启动后端、导入演示数据、启动 Next 前端，并走完整一期能力闭环。

  • 本地依赖：MySQL、Redis、MinIO
  • 核心路径：门户、人才服务区、求职者中心、企业后台、运营后台
  • 演示方式：显式导入 demo 数据 + 登录助手写入 localStorage
  • 安全边界：人才服务区只展示发布快照，对接接口仅 stub

本文面向第一次接手 LocalTalent 的新同事。照着做，你可以在本地拉起 MySQL/Redis/MinIO、启动后端、导入演示数据、启动 Next 前端，并看到一期已经具备的核心能力。

一、演示边界

  • 演示数据只用于本地开发和验收演示，不是生产初始化数据。
  • 演示数据通过 scripts/seed_demo_data 显式导入，不进入 Flyway 自动迁移，不影响 CI 空库迁移测试。
  • 人才服务区仍然只展示发布快照，不展示手机号、邮箱、简历正文、附件、证据等原始候选人数据。
  • 对接接口仍然只是 stub，不连接 ATS、受控候选池或外部主库。
  • 当前没有正式登录页，登录通过 scripts/demo_login_tokens 获取 JWT 并写入浏览器 localStorage。

二、你现在可以看到什么

  • 门户首页：Next/SSR 首页与基础入口。
  • 人才服务区：公开展示候选人发布快照，可按城市和岗位方向筛选。
  • 求职者中心：展示本人简历摘要、投递摘要、签到摘要、同意/撤回发布状态。
  • 企业后台：展示企业认证、职位状态、投递池、导出申请与下载入口逻辑。
  • 运营后台：展示企业审核、职位审核、导出审批与审计 trace 快速入口。
  • 导出审批：企业发起导出申请，operator 审批后由后端异步生成 CSV 并写入 MinIO。
  • 审计中心：按 trace_id 串联 audit_log / field_access_log / open_api_log。
  • 对接接口 stub：签名、幂等、trace、nonce 防重放和重试队列基础设施已具备。

三、本地环境检查

建议版本：

  • Docker Desktop 或 Docker Engine
  • JDK 21
  • Node 24 LTS
  • npm

检查命令：

docker --version
java -version
node -v
npm -v

如果 Docker 不可用，先打开 Docker Desktop。后端测试和本地依赖都需要 Docker。

四、启动依赖服务

在仓库根目录执行：

docker compose -f infra/docker-compose.yml up -d

这会启动三类本地依赖：

  • MySQL：业务数据库，端口 3306
  • Redis：一期本地占位缓存，端口 6379
  • MinIO：本地对象存储，用于导出文件和短期下载链接，端口 9000/9001

健康检查：

docker exec localtalent-redis redis-cli ping
docker exec -e MYSQL_PWD=localtalent localtalent-mysql mysql -ulocaltalent localtalent -e "SELECT 1;"
curl http://127.0.0.1:9000/minio/health/live

五、启动后端

打开一个终端：

cd backend
./mvnw spring-boot:run

第一次启动时，Flyway 会自动创建表并执行基础 seed。看到应用启动完成后，在另一个终端验证：

curl http://127.0.0.1:8080/health

也可以用打包方式启动：

cd backend
./mvnw clean package
java -jar target/backend-0.1.0-SNAPSHOT.jar

六、导入演示数据

后端至少启动过一次并完成 Flyway 迁移后，在仓库根目录执行：

./scripts/seed_demo_data

脚本会做四件事：

  • 检查 localtalent-mysql 容器是否在运行。
  • 检查核心表是否存在，避免在 Flyway 尚未迁移时导入。
  • 导入 demo/localtalent_demo_data.sql。
  • 扫描 candidate_publish_snapshot.snapshot_json，确认发布快照不包含原始候选人字段。

成功时会输出 PASS、演示账号、页面地址、待审批导出编号和可用筛选条件。

如需清空演示环境：

docker compose -f infra/docker-compose.yml down -v
docker compose -f infra/docker-compose.yml up -d
cd backend && ./mvnw spring-boot:run
./scripts/seed_demo_data

down -v 会删除本地 Docker volume，请只在开发演示环境使用。

七、启动前端

打开另一个终端：

cd frontend
npm ci
npm run dev

默认访问：

  • 前端：http://localhost:3000
  • 后端：http://localhost:8080

Next 服务端渲染默认通过 LOCALTALENT_API_BASE_URL=http://localhost:8080 访问后端。

九、登录方式

当前一期没有正式登录页。请用登录助手获取 token。

示例：企业后台登录。

./scripts/demo_login_tokens company

脚本会输出类似下面的浏览器 Console 命令：

localStorage.setItem('localtalent_access_token', '...');
localStorage.removeItem('localtalent_admin_role_hint');
location.href = '/company';

打开浏览器 DevTools Console，粘贴执行即可进入对应页面。

运营账号：

./scripts/demo_login_tokens operator

审计只读账号：

./scripts/demo_login_tokens auditor

求职者三种状态：

./scripts/demo_login_tokens candidate-consented
./scripts/demo_login_tokens candidate-revoked
./scripts/demo_login_tokens candidate-pending

退出登录：

localStorage.removeItem('localtalent_access_token');
localStorage.removeItem('localtalent_admin_role_hint');

十、推荐演示路径

1. 门户人才服务区

访问：

http://localhost:3000/portal/talent-service-area

输入筛选：

city_code=310000
category_code=software

你应该能看到 林** 的发布快照卡片。这里是公开展示区，只展示：

  • snapshot_id
  • display_name_masked
  • city_code
  • category_code
  • skills_summary
  • experience_years
  • updated_at

不会展示联系方式、简历正文、附件、证据等原始候选人数据。

2. 求职者中心

已同意账号：

./scripts/demo_login_tokens candidate-consented

访问 /candidate/center，应看到：

  • 简历完成度
  • 最近投递
  • 最近签到
  • 已同意/发布快照可见
  • 撤回入口

已撤回账号：

./scripts/demo_login_tokens candidate-revoked

应看到“已撤回并下线”的状态，人才服务区不会展示该候选人。

不可发布账号：

./scripts/demo_login_tokens candidate-pending

应看到不可发布/尚未有效同意状态。

3. 企业后台

./scripts/demo_login_tokens company

访问 /company，可以看到：

  • 企业认证提交区域
  • 职位状态列表：在线、待审核、下线
  • 投递池最小列表
  • 导出申请入口
  • 导出详情查询与下载入口显示逻辑

注意：未审批或未生成的导出不会显示下载按钮；后端也会拒绝未审批下载。

4. 运营后台

operator：

./scripts/demo_login_tokens operator

访问 /admin，可以看到：

  • 待审核企业
  • 待审核职位
  • 待审批导出
  • 审计 trace 查询入口

operator 可以点击审批按钮。导出审批通过后，后端会异步生成文件并写入 MinIO。稍等几秒后，企业后台用导出编号查询详情，生成完成且未发放过下载链接时会显示“获取下载链接”。

auditor：

./scripts/demo_login_tokens auditor

访问 /admin，只能看到只读队列和审计入口，写按钮不会显示；如果直接调用写接口，后端仍会返回 AUTHZ_403。

5. 审计 trace

在运营后台审计入口输入：

trace-demo

应能看到 audit、field access、open api 三类日志数量摘要。审计中心响应会脱敏敏感内容。

十一、MinIO 怎么看

MinIO 是本地对象存储，可以把它理解成“本地版 S3 文件仓库”。一期中主要用于导出文件：

  • 企业提交导出申请。
  • operator 审批通过。
  • 后端异步生成 CSV。
  • CSV 上传到 MinIO 的 export bucket。
  • 企业获取短期预签名下载链接。

访问控制台：

http://localhost:9001

登录：

用户名：localtalent
密码：localtalent123

审批通过并生成导出后，可以在 export bucket 里看到 CSV 对象。

十三、常见问题

scripts/seed_demo_data 提示核心表不存在

说明后端还没有跑过 Flyway 迁移。先启动后端：

cd backend
./mvnw spring-boot:run

看到启动完成后，再执行：

./scripts/seed_demo_data

登录助手失败

先确认后端可访问：

curl http://127.0.0.1:8080/health

如果后端没启动，先启动后端。若账号不存在，重新导入演示数据。

前端页面显示无权限

确认浏览器 Console 已执行登录助手输出的 localStorage.setItem 命令。运营后台还需要：

localStorage.setItem('localtalent_admin_role_hint', 'operator');

auditor 只读演示则使用：

localStorage.setItem('localtalent_admin_role_hint', 'auditor');

人才服务区没有数据

确认演示数据已导入，并使用筛选：

city_code=310000
category_code=software

也可以重新执行：

./scripts/seed_demo_data

导出审批通过后没有马上生成

导出是异步生成，稍等几秒后再用企业后台查询导出详情。如果仍未生成，检查后端日志和 MinIO 是否可用。

端口冲突

默认端口：

  • 后端：8080
  • 前端：3000
  • MySQL：3306
  • Redis：6379
  • MinIO API：9000
  • MinIO Console：9001

如果端口被占用，请先停止占用进程或调整配置。

十四、当前限制

  • 没有正式登录页，登录通过脚本写入 localStorage。
  • 求职者中心只做概览与同意/撤回入口，不做完整简历编辑 UI。
  • 企业后台没有完整职位创建 UI，只展示已有后端链路的最小演示入口。
  • 运营后台没有完整 CMS/event 前端页面。
  • 对接接口只是 stub，不做真实同步。
  • MinIO 是本地开发占位；生产对象存储、密钥轮换和生命周期策略需要后续专项治理。

十五、一句话理解一期

LocalTalent 一期已经具备“门户展示、求职者同意/撤回、企业职位与投递、运营审核、导出审批、对接 stub、审计追踪”的后端闭环和最小前端入口；演示数据负责把这些能力串成可看、可点、可讲解的本地故事线。
