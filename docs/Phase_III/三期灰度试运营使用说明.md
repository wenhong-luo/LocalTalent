# LocalTalent 三期灰度试运营使用说明

> 本说明面向第一次接手 LocalTalent 三期的人：怎么启动、怎么看页面、怎么打开三期能力、怎么做灰度前验收。三期目标是 **小范围真实试运营**，不是公网正式上线，也不是商业化完整版。

## 1. 先读这一段

- 三期主线 `Prompt 25-31` 已收口，当前可以进入灰度复核。
- 三期新增能力默认由 feature flag 控制，默认关闭。
- 公开门户仍继承一二期边界：人才服务区只展示发布快照，公开层不展示原始候选人数据。
- 对接接口仍是 stub；真实 ATS/RPA、受控候选池同步不在三期。
- 真实短信、微信、小程序、App、地图、视频、直播、支付、会员商业化、公共简历库、受控找人才、联系解锁都不在三期主线。

## 3. 本地环境准备

启动基础依赖：

```bash
docker compose -f infra/docker-compose.yml up -d
```

启动后端：

```bash
cd backend
./mvnw spring-boot:run
```

启动前端：

```bash
cd frontend
npm ci
npm run dev
```

默认访问地址：

| 服务 | 地址 |
| --- | --- |
| 前端 | `http://localhost:3000` |
| 后端健康检查 | `http://localhost:8080/health` |
| MySQL / Redis / MinIO | 使用 `infra/docker-compose.yml` 中的本地配置 |

## 4. 导入演示数据

三期没有新增生产 seed。建议先导入二期高保真门户演示数据，再用三期页面和灰度脚本验证：

```bash
./scripts/seed_phase2_demo_data
```

演示数据只用于本地，不进入 Flyway，不作为生产初始化数据。

## 5. 三期 feature flag

三期 flag 见：

```bash
docs/Phase_III/feature_flags.md
```

本地如需查看三期私有闭环，可在启动后端前显式设置：

```bash
export LOCALTALENT_PHASE3_CANDIDATE_CLOSURE=true
export LOCALTALENT_PHASE3_COMPANY_WORKBENCH=true
export LOCALTALENT_PHASE3_OPERATOR_PORTAL_OPS=true
export LOCALTALENT_PHASE3_COMPANY_STYLE_UPLOAD=true
```

注意：

- 不设置时，三期新增业务能力默认关闭，页面会保留一二期基础视图或提示能力未开启。
- `gray` 环境不能随意打开全部 flag，必须按白名单、比例和观察窗口逐步开启。
- flag 不得绕过 RBAC、数据域、字段级权限、导出审批或审计。

## 7. 页面 walkthrough

### 7.1 公开门户

| 页面 | 地址 | 看什么 |
| --- | --- | --- |
| 首页 | `/` | 高保真门户、推荐位、公开频道入口 |
| 找工作 | `/jobs` | 公开职位搜索与职位卡片 |
| 名企直聘 | `/jobs/famous` | 名企/认证企业相关职位入口 |
| 急聘/快速招聘 | `/jobs/emergency` | 低风险公开职位专题 |
| 今日招聘 | `/daily-jobs` | 近期更新职位专题 |
| 找企业 | `/companies` | 已认证企业公开列表 |
| 企业主页 | `/companies/[id]` | 企业公开简介与在线职位 |
| 人才服务区 | `/portal/talent-service-area` | 只展示发布快照白名单 |
| 招聘会 | `/job-fairs` | 公开招聘会列表 |
| 网络招聘会 | `/job-fairs/online` | 网络招聘会公开入口 |
| 校园招聘 | `/campus` | 校园职位、校园活动、校园资讯聚合 |
| 就业政策 | `/articles/policies` | 政策内容公开频道 |
| HR 工具箱 | `/hr-tools` | HR 工具公开频道 |
| 更多服务 | `/more-services` | 低风险入口与高风险占位 |

公开页面不得出现手机号、邮箱、简历正文、附件对象 key、证据、原始 JSON、审核材料、报名名单、签到证据。

### 7.2 求职者中心

地址：

```text
/candidate/center
```

三期开启 `LOCALTALENT_PHASE3_CANDIDATE_CLOSURE=true` 后可查看：

- 首次完善简历流程：`/candidate/resume/create`，覆盖完善简历 1、完善简历 2、创建完成三页。
- 求职者个人中心首页：`/candidate/center` 已增强为接近 `docs/page/4、会员中心.png` 的高保真个人中心布局，包含个人横幅、统计卡、左侧菜单、简历状态卡、优选服务占位、职位推荐区和右侧客服占位。
- 简历编辑与预览
- 投递记录
- 职位收藏
- 搜索订阅
- 站内通知
- 隐私、同意、撤回与发布状态
- 简历附件上传、下载、替换、删除（仅本人私有域，需开启 `LOCALTALENT_PHASE3_RESUME_ATTACHMENT_UPLOAD=true`）
- 简历智能优化建议（安全规则版，本地规则生成、逐条手动应用，需开启 `LOCALTALENT_PHASE3_RESUME_AI_ASSIST=true`）

关键边界：

- 只能访问本人数据。
- 首次完善简历进度由服务端 `candidate_resume_onboarding` 表判断，前端不得用 localStorage 自行推断。
- 写操作带 `X-Trace-Id` 与 `X-Idempotency-Key`。
- 撤回后人才服务区与公开推荐位不可见。
- 私有中心 `noindex`，不进入 sitemap。
- “会员中心”只作为个人中心视觉参考，不代表会员商业化、真实支付、联系解锁或公共简历库；优选服务中的商业化项均为灰度占位。
- 当前不做附件解析、真实外部 AI/LLM、短信、微信、小程序或 App。
- 附件 object key 仅服务端私有保存；前端、公开门户、人才服务区、日志明文、sitemap、搜索和导出旁路均不得出现 object key、预签名 URL 或文件内容。
- AI 优化建议只使用服务端规范化 DTO 和本地规则；不上传原始候选人数据、不保存完整 prompt 或模型原文、不自动发布到人才服务区。

### 7.3 企业中心

地址：

```text
/company
```

三期开启 `LOCALTALENT_PHASE3_COMPANY_WORKBENCH=true` 后可查看：

- 企业工作台首页
- 企业资料维护
- 认证材料提交
- 职位创建、编辑、送审、下线
- 投递池与投递详情
- 阶段流转
- 面试邀约
- 导出申请入口与状态
- 企业风采私有图片上传、预览、排序、删除（仅企业私有域，需开启 `LOCALTALENT_PHASE3_COMPANY_STYLE_UPLOAD=true`）
- 企业基本资料 Logo 私有上传、预览、替换、删除（仅企业私有域，需开启 `LOCALTALENT_PHASE3_COMPANY_LOGO_UPLOAD=true`）

关键边界：

- 企业只能访问本企业数据。
- 企业风采图片只在 `/company` 企业管理 > 企业风采中私有可见；object key 仅服务端保存，不返回前端、不进入公开企业主页、推荐位、搜索、sitemap、日志明文或导出旁路。
- 企业 Logo 只在 `/company` 企业管理 > 基本资料中私有可见；object key 仅服务端保存，不返回前端、不进入公开企业列表、企业公开主页、推荐位、搜索、sitemap、日志明文或导出旁路。
- 投递池不是公共简历库。
- 不展示未授权手机号、邮箱、简历正文、附件对象 key、证据或原始 JSON。
- 导出仍必须走审批、字段级权限、短期下载和审计链。

### 7.4 运营后台

地址：

```text
/admin
```

三期开启 `LOCALTALENT_PHASE3_OPERATOR_PORTAL_OPS=true` 后可查看：

- 运营首页
- 企业审核
- 职位审核
- 内容/活动/招聘会管理
- 推荐位配置
- 风险审核
- 审计中心 trace 查询

关键边界：

- `operator` 可在授权范围内写操作。
- `auditor` 只读，写操作必须被服务端拒绝。
- 推荐位只能引用已审核、已发布、在线、未撤回且字段白名单裁剪后的对象。
- 审计响应必须脱敏。

## 8. 灰度前必跑命令

静态与边界闸门：

```bash
./scripts/check_phase3_boundary
./scripts/check_phase3_ops
./scripts/check_phase3_security_acceptance
./scripts/check_phase3_gray_acceptance
./scripts/check_portal_snapshot_fields
./scripts/check_boundary
```

全量总闸门：

```bash
./scripts/run_all
```

运行中应用 smoke：

```bash
./scripts/smoke_phase3
```

公开读容量烟测：

```bash
./scripts/load_phase3_public
```

备份恢复演练：

```bash
./scripts/backup_phase3_mysql
LOCALTALENT_PHASE3_BACKUP_FILE=/tmp/localtalent-phase3-backups/<dump>.sql ./scripts/restore_phase3_mysql_drill
```

如果 `load_phase3_public` 首次因为本地 Next 冷启动导致 P95 超阈值，但所有请求均为 200，可热身后复跑一次；灰度环境仍应以稳定复跑结果和监控指标为准。

## 10. 当前限制

三期仍明确不做：

- 公共简历库。
- 搜索简历、查看完整简历。
- 受控找人才。
- 联系解锁。
- 会员商业化和真实支付。
- 真实短信、微信、小程序、App 推送。
- 真实地图、视频、直播。
- 真实 ATS/RPA 或受控候选池同步。
- AI 处理原始候选人数据。
- 公网正式运营 SLA。

这些能力必须后续单独走风险准入或四期/专项规划，不能混入三期已收口主线。

## 11. 一句话理解三期

三期把 LocalTalent 从“能演示的人才网”推进到“可小范围真实试运营的人才网”：求职者、企业、运营人员可以在受控开关和审计边界内真实使用，但系统仍不开放高风险能力，也不等同于公网正式上线。
