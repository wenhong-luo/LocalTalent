1. 执行摘要

三期不是继续堆静态页面，也不是直接冲完整商业化。三期的核心任务是把一期的合规安全底座和二期的高保真门户，推进为一套真实用户可灰度试运营的业务闭环。

  • candidate 能注册登录、维护简历、投递、收藏、订阅、接收站内通知、同意与撤回。
  • company 能维护企业资料、提交认证、发布职位、处理投递、邀约面试、申请导出。
  • operator 能审核、配置、发布、下线、推荐、处理风险和查审计。
  • auditor 能只读查看运营和审计数据，不具备任何写权限。
  • 公开门户由真实审核状态、发布状态和运营配置驱动。
  • 灰度试运营具备预发配置、feature flag、备份恢复、监控告警、容量烟测和回滚 Runbook。

5. 任务拆解

5.1 基础治理与灰度闸门

  • 建立三期 README、环境模板、feature flag 注册表、开发日志和边界脚本。
  • 所有三期 flag 默认关闭，必须能按用户范围、角色范围和功能范围灰度打开。
  • 增加三期边界与运维闸门，阻断风险池能力、敏感字段和越界文件。

5.2 认证与身份映射

  • 预发/灰度默认 OIDC；本地账号 + JWT 仅用于 dev/test 和应急白名单。
  • IdP 只负责“你是谁”，LocalTalent 后端继续负责“你能看什么、能做什么”。
  • 日志不得输出 token、secret、claims 明文。

5.3 求职者中心生产化

  • 完成简历编辑、预览、投递记录、收藏、订阅、站内通知、同意/撤回和发布状态。
  • 状态全部由服务端返回，前端不得推断。
  • 撤回后必须联动发布快照、人才服务区、推荐位、搜索索引、缓存和 sitemap。

5.4 企业招聘工作台生产化

  • 完成企业资料维护、认证材料提交、职位创建/编辑/送审/下线、投递池、投递详情、面试邀约和导出申请入口。
  • 企业只能访问本企业资料、职位、投递、面试和导出申请。
  • 未认证企业不得上线职位；未审核职位不得进入公开门户。

5.5 运营后台与门户运营化

  • operator 能审核企业、职位、内容、活动，管理招聘会、推荐位、风险审核和审计入口。
  • auditor 只读，不得调用任何写接口。
  • 公开门户由真实状态和运营配置驱动。

5.6 安全风控与工程灰度

  • 核心业务动作按 trace 串联 audit、field access、open api。
  • 导出继续走审批、异步生成、字段裁剪、短期下载和审计链。
  • 建立预发、灰度、备份恢复、监控告警、容量烟测和 Runbook。

6. 接口、数据与字段边界

本蓝图不定义最终 API schema。后续每个 Prompt 必须在当前阶段内细化接口，并遵守统一响应体、X-Trace-Id、服务端 RBAC、数据域、字段级权限、必要幂等和审计要求。

身份可访问数据必须阻断
visitor公开职位、公开企业、公开内容、公开活动、发布快照白名单。candidate/company/admin 私有接口和原始候选人数据。
candidate本人简历、投递、收藏、订阅、通知、同意/撤回状态。其他 candidate 数据、company 私有数据、admin 数据。
company本企业资料、职位、投递池、面试、导出申请。其他 company 数据、未授权候选人字段、公共简历库能力。
operator授权范围内审核、配置、风险处理和审计查询。绕过审核、字段级权限、导出审批或审计留痕。
auditor只读审核、运营、风险和审计数据。任何写接口。
openapi client签名、nonce、幂等保护下的 stub 契约。原始候选人数据、真实同步、绕过签名或重放。

禁止进入公开响应、公开 DOM、搜索索引、缓存、sitemap、日志、埋点和审计响应的字段包括手机号、邮箱、联系方式、简历正文、附件对象 key、证据、原始候选人 JSON、审核材料、后台备注、报名名单、签到证据、secret、token、signature、nonce 明文。

7. 迁移、配置与 feature flag 纪律

  • 不修改旧迁移；任何数据库结构变化只能新增 forward-only migration。
  • DB 结构问题优先 roll-forward 修复，不直接回滚已应用结构。
  • .env.phase3.example 只能保存示例值，不保存真实密钥。
  • local、staging、gray 的配置差异必须可读、可审计。
  • 三期新增能力默认关闭，每个 flag 必须有负责人、默认值、影响范围、灰度范围和回滚方式。
  • flag 不得用于绕过服务端权限、字段级权限、导出审批或审计留痕。

9. 灰度试运营进入条件、暂停条件和退出条件

进入条件

  1. 当前阶段范围、禁止事项、feature flag 和上线对象已经冻结。
  2. 边界脚本、后端测试、前端测试、构建和三期专项脚本通过。
  3. OIDC 或本地 dev 登录正常，跨对象、跨企业、跨角色访问被后端拒绝。
  4. candidate、company、operator、auditor 关键闭环跑通。
  5. 导出审批、字段访问留痕、trace 串联和日志脱敏通过。
  6. 人才服务区、公开搜索、sitemap、缓存、埋点没有原始候选人数据旁路。
  7. 备份恢复、回滚、灰度开关、容量烟测、告警联通都跑过一遍。

暂停条件

  • 公开门户 API 5xx 持续超过 0.5%。
  • BOLA/IDOR、字段泄露、导出绕行、审计缺失等 P0/P1 安全问题。
  • 撤回下线传播超过 5 分钟仍未收敛。
  • 导出链接过期、二次发放或审批规则失效。
  • 任一核心身份无法登录或分流错误。

退出条件

  • 至少 20 位 candidate、10 家 company、3 名 operator/auditor 完整跑通真实闭环。
  • 公开门户真实状态驱动稳定，SEO 与 sitemap 不含私有或撤回对象。
  • 审计、导出、撤回、IDOR/BOLA、字段黑名单回归稳定。
  • 备份恢复、回滚演练、告警联通和容量烟测都有记录。
  • 风险池能力仍未偷跑。
