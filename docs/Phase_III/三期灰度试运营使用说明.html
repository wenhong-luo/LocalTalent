LocalTalent 三期灰度试运营使用说明

1. 先读这一段

三期主线 Prompt 25-31 已收口，目标是小范围真实试运营，不是公网正式上线。
  • 三期新增能力默认由 feature flag 控制，默认关闭。
  • 人才服务区只展示发布快照，公开层不展示原始候选人数据。
  • 对接接口仍是 stub；真实 ATS/RPA、受控候选池同步不在三期。
  • 真实短信、微信、小程序、App、地图、视频、直播、支付、会员商业化、公共简历库、受控找人才、联系解锁都不在三期主线。

2. 本地启动

docker compose -f infra/docker-compose.yml up -d
cd backend
./mvnw spring-boot:run
cd frontend
npm ci
npm run dev
服务地址
前端http://localhost:3000
后端健康检查http://localhost:8080/health

推荐先导入二期门户演示数据：

./scripts/seed_phase2_demo_data

3. Feature Flag

三期 flag 注册表：docs/Phase_III/feature_flags.md。

export LOCALTALENT_PHASE3_CANDIDATE_CLOSURE=true
export LOCALTALENT_PHASE3_COMPANY_WORKBENCH=true
export LOCALTALENT_PHASE3_OPERATOR_PORTAL_OPS=true
gray 环境不能随意打开全部 flag，必须按白名单、比例和观察窗口逐步开启。flag 不得绕过 RBAC、数据域、字段级权限、导出审批或审计。

私有中心

  • /candidate/resume/create：首次完善简历 1、完善简历 2、创建完成三页；进度由服务端 candidate_resume_onboarding 表判断；开启 LOCALTALENT_PHASE3_RESUME_ATTACHMENT_UPLOAD=true 后支持本人私有简历附件上传、下载、替换、删除；开启 LOCALTALENT_PHASE3_RESUME_AI_ASSIST=true 后支持安全规则版简历优化建议。
  • /candidate/center：已增强为接近 docs/page/4、会员中心.png 的求职者个人中心首页，包含个人横幅、统计卡、左侧菜单、简历状态卡、优选服务占位、职位推荐区、简历、投递、收藏、订阅、站内通知、智能优化建议、同意/撤回。
  • /company：企业资料、认证、职位、投递池、面试邀约、导出申请。
  • /admin：审核、推荐位、风险审核、审计 trace。

首次完善简历和个人中心均属于 candidate 本人私有域；“会员中心”只作为个人中心视觉参考，不代表会员商业化、真实支付、联系解锁或公共简历库，优选服务中的商业化项均为灰度占位。当前不做附件解析、真实外部 AI/LLM、短信、微信、小程序或 App。附件 object key 仅服务端私有保存，不返回前端、不进入公开门户、人才服务区、日志明文、sitemap、搜索或导出旁路。AI 优化建议只使用服务端规范化 DTO 和本地规则，不上传原始候选人数据、不保存完整 prompt 或模型原文、不自动发布到人才服务区。

7. 当前限制

  • 不做公共简历库、搜索简历、查看完整简历。
  • 不做受控找人才、联系解锁、会员商业化和真实支付。
  • 不接真实短信、微信、小程序、App、地图、视频、直播。
  • 不接真实 ATS/RPA 或受控候选池同步。
  • 不承诺公网正式运营 SLA。
一句话理解三期：从“能演示的人才网”推进到“可小范围真实试运营的人才网”。
