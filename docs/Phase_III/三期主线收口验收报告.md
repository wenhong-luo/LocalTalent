## 4. 已落实的安全边界

- 人才服务区只展示 `candidate_publish_snapshot` 发布快照。
- `candidate_resume_onboarding` 只保存首次完善简历流程状态，不保存手机号、邮箱、简历正文、附件对象 key、AI 文本、证据或原始 JSON。
- 公开层不输出手机号、邮箱、简历正文、附件对象 key、证据、原始 JSON、审核材料、报名名单、签到证据。
- candidate 私有接口只能访问本人数据。
- company 私有接口只能访问本企业数据。
- operator 写操作要求权限、trace、幂等和审计。
- auditor 只读，写操作被服务端拒绝。
- 导出仍走 `export_apply -> operator 审批 -> 异步生成 -> MinIO/S3 短期 URL -> audit_log/field_access_log`。
- OIDC 只负责认证和基础 claims，RBAC、数据域、字段级权限仍由 LocalTalent 服务端强制。
- OpenAPI 对接仍为 stub，不接真实 ATS/RPA 或外部主库。

以下能力未实现，也不得通过配置、脚本、推荐位、缓存、备注或灰度开关偷跑：

- 公共简历库。
- 受控找人才。
- 联系解锁。
- 会员商业化。
- 真实支付。
- 真实短信、微信、小程序、App。
- 真实地图、视频、直播。
- 真实 ATS/RPA 或受控候选池同步。
- AI 处理原始候选人数据。
