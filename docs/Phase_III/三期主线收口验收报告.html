4. 已落实的安全边界

  • 人才服务区只展示发布快照。
  • 公开层不输出手机号、邮箱、简历正文、附件对象 key、证据、原始 JSON、审核材料、报名名单、签到证据。
  • candidate/company 私有接口强制本人/本企业数据域。
  • operator 写操作要求权限、trace、幂等和审计；auditor 只读。
  • 导出仍走审批、字段裁剪、短期下载和审计链。
  • OIDC 只负责认证，RBAC 和字段级权限仍由服务端强制。
  • OpenAPI 对接仍为 stub。

未进入三期主线：公共简历库、受控找人才、联系解锁、会员商业化、真实支付、真实短信/微信/小程序/App、真实地图/视频/直播、ATS/RPA、AI 处理原始候选人数据。
