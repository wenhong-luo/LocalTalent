二、一二期继承基线

能力域一期/二期已完成事实三期继承口径
合规安全底座RBAC、数据域、字段级权限、同意/撤回、审计、trace、导出审批、对接 stub 已建立。不重做底座；所有新增写动作和对象 ID 都必须继续走服务端强制。
人才服务区公开数据源唯一为 candidate_publish_snapshot。继续只展示发布快照白名单；撤回后不可见。
公开门户二期已完成首页、找工作、找企业、招聘会、资讯、HR 工具箱、P1 公开频道。三期让这些页面从演示/静态数据转为真实状态和后台配置驱动。
求职者中心已有 overview、同意/撤回链路、投递/签到基础链路。三期补真实简历维护、投递记录、收藏订阅、通知和状态回流。
企业中心已有企业认证、职位状态、投递池、导出申请入口基础能力。三期补企业资料、认证材料、职位发布编辑、投递处理和邀约工作台。
运营后台已有审核、CMS/event、导出审批、审计入口。三期补可支撑试运营的配置、风险处理、推荐位和审计 UI。
三期的关键词是 真实试运营闭环，不是“更多 demo 页面”。

三、三期总目标

把一二期的可演示系统，推进为真实求职者、企业、运营人员可小范围灰度试用的业务系统。
  1. 真实求职者能不能注册、维护简历、投递、看状态、收藏订阅、接收通知、撤回同意。
  2. 真实企业能不能认证、维护资料、发布职位、处理投递、邀约面试、申请导出。
  3. 真实运营能不能审核、配置、处理风险、查审计，并支撑公开门户持续运营。

三期不是公网正式运营。公网正式运营所需的压测、客服工单、合规材料、生产级监控告警和完整上线演练，建议放到四期。

四、灰度试运营定义

角色灰度试运营最低可用标准
candidate可注册登录、编辑简历、预览简历、投递职位、查看投递/面试状态、收藏职位、订阅搜索、接收站内通知、同意/撤回发布。
company可注册登录、维护企业资料、提交认证材料、创建/编辑职位、提交审核/下线、处理投递、备注阶段、邀约面试、申请导出。
operator可审核企业、职位、内容、活动，配置首页/频道推荐位，处理风险，查看审计。
auditor可只读查看审计、风险和运营记录，不得执行任何写操作。
visitor可浏览真实状态驱动的公开门户，但看不到原始候选人数据、联系方式、报名名单、审核材料。

灰度试运营必须保留可回滚、可审计、可停止的能力。如果某个功能不能安全回滚，就不应进入三期主线。

五、三期主线能力域

5.1 求职者真实闭环

  • 简历编辑与预览。
  • 隐私与发布状态。
  • 投递记录与状态回流。
  • 收藏与订阅。
  • 站内通知。
  • 同意/撤回便捷操作。
  • 撤回后人才服务区不可见回归。
求职者中心是本人私有域；不得把手机号、邮箱、简历正文、附件对象 key、证据、原始 JSON 带入公开门户、搜索索引、缓存、日志或导出。

5.2 企业招聘工作台

  • 企业资料维护。
  • 企业认证材料提交。
  • 职位创建、编辑、提交审核、下线。
  • 投递池筛选、详情、备注、阶段流转。
  • 面试邀约。
  • 导出申请入口。
企业只能访问本企业数据；未认证企业不得上线职位；投递池不是公共简历库；所有对象 ID 接口必须覆盖 IDOR/BOLA 测试。

5.3 运营后台生产化

  • 企业审核、职位审核、内容与活动管理、招聘会公开展示配置。
  • 首页与频道推荐位、风险审核、审计中心 UI。

operator 可写，auditor 只读；审核、下线、软删除、推荐位调整和风险处理必须写 audit_log。

5.4 门户运营化

  • 首页推荐从静态/演示数据转为后台配置。
  • 找工作、找企业、招聘会、资讯、HR 工具箱使用真实状态。
  • 人才服务区继续只读发布快照。

5.5 工程试运营治理

  • 预发/生产环境变量说明、备份恢复演练、对象存储生命周期策略。
  • 日志脱敏检查、基础监控告警、上线/回滚 Runbook、真实数据导入前检查单。

六、三期明确不做

能力三期结论说明
公共简历库禁止人才服务区仍只展示发布快照。
受控找人才暂缓必须先走风险准入，完成授权、权限、审计和撤回联动方案。
联系解锁暂缓涉及联系方式、高敏字段和权益/审批。
会员商业化/真实支付暂缓涉及资金流、对账、发票、退款和风控。
真实短信/微信/小程序/App暂缓三期默认只做站内通知。
真实地图/视频/直播暂缓只保留占位或风险池入口。
真实 ATS/RPA/受控候选池同步暂缓对接接口继续 stub。
AI 处理原始候选人数据暂缓需要最小必要、脱敏、供应商审查和人工复核。

七、阶段拆分继承口径

LocalTalent_三期总规划.md 已给出 P3-0 至 P3-6 的作战手册。Step1 不重复展开 Prompt，只冻结继承口径。

阶段Step1 继承口径
P3-0 三期基线冻结与灰度闸门只做边界、脚本、flag、env、README，不做业务功能。
P3-1 认证与身份映射预发/灰度默认 OIDC，本地/开发保留本地账号 + JWT；授权仍由平台服务端强制。
P3-2 求职者真实闭环生产化优先实现求职者中心真实可用，包含简历、投递、收藏、订阅、通知、同意/撤回。
P3-3 企业招聘工作台生产化实现企业认证材料、职位发布、投递池处理、面试邀约、导出申请。
P3-4 运营后台生产化与门户运营化实现运营审核配置、推荐位、内容活动管理、审计 UI 和真实状态驱动门户。
P3-5 审计/导出/风控/安全收口强化撤回下线、字段泄露、导出审批、审计 trace、BOLA/IDOR。
P3-6 灰度试运营工程治理与上线演练补备份恢复、容量烟测、回滚演练、灰度 KPI 和 Runbook。

八、自动化验收关注点

  • 人才服务区字段白名单。
  • 撤回后下线联动。
  • 公开页面字段黑名单。
  • RBAC 与数据域。
  • IDOR/BOLA。
  • audit_log 与 field_access_log。
  • 统一响应体与 trace_id。
  • 写接口幂等。
  • auditor 只读。
  • 风险池能力不得进入主线。
  • 公开门户、搜索索引、sitemap、缓存、日志、埋点不得包含原始候选人数据。

九、Step1 结论与下一步

  • 三期目标明确为“小范围真实试运营闭环”。
  • 三期不直接做公网正式运营，也不做完整商业化。
  • 三期主线优先补真实求职者、企业、运营工作流和试运营工程治理。
  • 三期研发方向与上线阶段判断.md 保留为决策备忘录。
  • LocalTalent_三期总规划.md 是三期主规划和作战手册。
  • 高风险能力继续留在风险池，必须先走风险准入。
