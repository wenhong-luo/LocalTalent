## 4. 暂停步骤

1. 关闭相关 feature flag。
2. 暂停扩大灰度比例。
3. 暂停高风险写动作，例如导出申请、职位发布、推荐位调整。
4. 执行边界脚本与 smoke。
5. 记录 trace_id、影响范围和处置动作。

## 5. 回滚步骤

1. 关闭相关 feature flag。
2. 若仍异常，回滚应用版本。
3. 数据库结构不做破坏性回滚，优先 roll-forward 修复。
4. 执行 `./scripts/smoke_phase3`。
5. 执行 `./scripts/check_portal_snapshot_fields` 与 `./scripts/check_phase3_security_acceptance`。
6. 填写灰度故障记录。

## 6. 退出试运营判断

- [ ] 灰度连续稳定运行不少于 7 天。
- [ ] 核心 KPI 达标。
- [ ] 无 P0/P1 安全问题。
- [ ] 备份恢复、回滚演练、告警联通和容量烟测都有记录。
- [ ] 真实 candidate/company/operator/auditor 关键路径均完成验收。
