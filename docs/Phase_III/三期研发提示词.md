# LocalTalent 三期 Codex 投喂手册（可执行版 · Prompt 25–31）

> 用途：三期代码开发前必读 Markdown 版；打开对应 HTML 后可逐轮复制 Prompt 25–31。
> 重要：三期目标是“小范围真实试运营”，不是公网正式运营，也不是商业化完整版。

## 开发前必读卡

- **文件定位**：Codex 三期逐轮投喂执行手册，覆盖 Prompt 25–31，对应 P3-0 至 P3-6。
- **实现约束**：每轮只做当前 Prompt；不得跨 Prompt 偷跑；不得修改一期 Prompt 0–14、二期 Prompt 15–24 既定事实。
- **三期目标**：把一二期可演示系统推进为真实求职者、企业、运营人员可小范围灰度试运营的业务系统。
- **禁止越界点**：禁止公共简历库；禁止受控找人才、联系解锁、会员支付、真实短信、真实微信、小程序、App、真实地图、视频、直播、真实 ATS/RPA 或受控候选池同步。
- **自动化验收关注点**：人才服务区发布快照、字段黑名单、RBAC、数据域、字段级权限、IDOR/BOLA、审计 trace、导出审批、撤回下线、feature flag、灰度 Runbook。

**统一硬边界提醒**：人才服务区仅展示发布快照；原始候选人数据不得公开；对接接口仅 stub；同意、撤回、数据域、字段级权限、审计和导出审批必须由服务端收口。

## 失败情形处理矩阵

| 现象 | 优先判断 | 动作 |
| --- | --- | --- |
| 风险能力偷跑 | 是否出现公共简历库、联系解锁、真实短信/微信/App、支付、地图/视频/直播、ATS/RPA。 | 删除真实能力，保留占位或风险池文案；补边界脚本。 |

| IDOR/BOLA | 对象级授权或数据域缺失。 | 在 service 层补数据域校验，补跨 candidate、跨 company、跨角色测试。 |

## Prompt 26（P3-1）：认证与身份映射

主控说明

- Codex 必读：三期边界、三期总规划、Step1、三期蓝图、Prompt 25 完成记录。
- OIDC 只负责认证和基础 claims；LocalTalent 后端继续负责 RBAC、数据域、字段级权限和审计。
- 本轮只实现认证与身份映射，不实现求职者、企业、运营业务增强。

本轮目标

- 接入 OIDC/SSO 登录、回调、登出、失败态和环境配置。
- 建立 OIDC 外部身份到 LocalTalent 内部 candidate/company/operator/auditor 的映射策略。
- 保留本地账号 + JWT 作为 dev/test fallback；预发/灰度默认 OIDC，本地登录默认关闭或仅应急白名单启用。
- 日志脱敏：不得记录 token、secret、明文 claims、id_token、access_token、refresh_token。
- 前端登录入口按环境展示 OIDC 或本地 dev fallback，并保持无权限态一致。

禁止事项

- 禁止把角色授权完全交给 IdP claim；服务端仍必须强制权限和数据域。
- 禁止泄露 token、secret、claims 明文到日志、审计、前端错误、URL query。
- 禁止接入真实微信、短信、小程序、App、第三方社交登录。
- 禁止修改求职者/企业/运营业务流程。

输出要求（文件与命令）

- 后端：OIDC 配置、回调处理、内部账号映射、角色映射、登出、fallback 策略。
- 前端：登录入口适配、失败态、无权限态、trace_id 展示。
- 配置/文档：环境变量示例、local/staging/gray 策略、token 日志脱敏说明。
- 测试：OIDC 回调映射、本地 fallback、角色映射、token 不入日志、非授权角色阻断。
- 命令：`cd backend && ./mvnw test`、`cd frontend && npm test && npm run build`、三期边界脚本。

- [ ] 预发/灰度模式 OIDC 登录成功并映射到内部身份。
- [ ] dev/test 本地 JWT fallback 可用，gray/prod 默认关闭或白名单启用。
- [ ] candidate/company/operator/auditor 分流正确，权限仍由服务端强制。
