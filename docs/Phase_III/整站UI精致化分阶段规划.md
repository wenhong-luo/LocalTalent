# LocalTalent 整站 UI 精致化分阶段规划

## Summary

整站 UI 精致化建议拆成 **8 个提示词** 完成。当前问题不是单页 bug，而是控件高度、字体大小、卡片间距、筛选区布局、按钮风格、后台工作台密度不统一。

执行顺序建议为：先做统一视觉规范和共享控件，再逐个业务域套用。这样不会把视觉优化误伤成业务改造，也更容易分轮验收。

## 已完成记录

- `Prompt UI-1：全站视觉基线与共享控件收口` 已完成。
- UI-1 只改前端视觉底座和共享选择器样式，不改后端、不新增接口、不新增迁移、不改变权限或业务边界。
- `Prompt UI-2：招聘者后台 Shell 精致化` 已完成。
- UI-2 只优化 `/company` 顶部栏、左侧菜单、内容区宽度、背景、卡片密度和响应式，不改接口、状态机、门禁、权限或风险能力。
- `Prompt UI-3：搜索简历 UI 高保真重做` 已完成。
- UI-3 只优化 `/company` 搜索简历页筛选区、简历卡片、分页和空态视觉，不改搜索接口、详情/举报/申请门禁或字段黑名单。
- `Prompt UI-4：职位管理与发布职位 UI 精致化` 已完成。
- UI-4 只优化 `/company` 职位管理列表、发布/编辑职位长表单、已删除回收站、批量操作栏、tab、表格和分页样式，不改职位接口、状态机、软删除/恢复逻辑或风险能力。
- `Prompt UI-5：企业管理与会员首页 UI 精致化` 已完成。
- UI-5 只优化 `/company` 企业基本资料、Logo 私有上传卡、企业风采私有管理页和招聘者会员首页视觉，不改 `profile / logo / style-images` 接口，不把 Logo 或企业风采接入公开企业主页。
- `Prompt UI-6：求职者端 UI 精致化` 已完成。
- UI-6 只优化 `/candidate/resume/create`、`/candidate/center`、期望职位/期望地区选择器、附件上传区和 AI 建议卡片视觉，不改 candidate 私有接口、附件上传逻辑、AI 安全规则版逻辑或发布快照边界。
- 已通过前端测试、构建和三期边界脚本。

## 推荐 8 轮提示词

### Prompt UI-1：全站视觉基线与共享控件收口

- 建立统一 UI tokens：字体、字号、行高、圆角、边框、阴影、间距、表单高度、按钮尺寸。
- 重构共享选择器 `DictionarySelect / RegionCascadePicker / ExpectedPositionPicker` 的紧凑风格。
- 目标：后续所有页面的输入框、下拉框、按钮、标签、卡片先统一“精致小巧”的底座。

### Prompt UI-2：招聘者后台 Shell 精致化

- 优化 `/company` 整体框架：顶部栏、左侧菜单、内容宽度、背景、卡片密度。
- 对齐 74CMS 后台风格：侧栏稳定、内容区自适应、控件小巧、留白合理。
- 不改业务功能，只改布局和样式。

### Prompt UI-3：搜索简历 UI 高保真重做

- 重点修复搜索简历页控件不对齐、控件过大、筛选行密度不统一、简历卡片不够精致的问题。
- 参考 `docs/page/招聘者会员/搜索简历/`：顶部地区/职位类型/关键词搜索条、学历/经验快捷筛选、六个下拉筛选、简历列表卡片。
- 画像分析、下载、联系、职聊继续保持受控占位。

### Prompt UI-4：职位管理与发布职位 UI 精致化

- 优化职位管理列表、发布职位长表单、已删除/回收站、状态按钮。
- 统一表格、批量操作栏、tab、分页、长表单分区样式。
- 不新增职位功能，只做视觉与交互一致性。

### Prompt UI-5：企业管理与会员首页 UI 精致化

- 优化企业基本资料、Logo 上传、企业风采、招聘者会员首页。
- 统一表单行距、上传卡片、右侧 Logo 区、提示弹窗、占位服务卡片。
- 保持企业风采/Logo 仍为企业私有域，不接公开展示。

### Prompt UI-6：求职者端 UI 精致化

- 优化求职者首次完善简历 1-3、会员中心、期望职位/期望地区选择器、附件上传、AI 建议卡片。
- 统一候选人私有中心的卡片、表单、步骤条、按钮和弹窗。
- 不改变人才服务区发布快照边界。

### Prompt UI-7：公开门户 UI 对齐

- 优化首页、找工作、找企业、人才服务区、招聘会、资讯、HR 工具箱、更多服务。
- 修复导航高亮问题，例如 `/companies` 时“找企业”必须高亮。
- 调整公开页面左右广告位/留白布局，使“找企业”与“首页/找工作”一致。

### Prompt UI-8：全站验收、响应式和文档收口

- 做统一回归：桌面宽屏、普通笔记本、移动端兜底。
- 补前端测试：导航高亮、选择器关闭、字段黑名单、私有页 `noindex`、风险占位不偷跑。
- 更新三期演示说明和领导演示路线，记录 UI 精致化完成范围。

## 推荐优先级

- 如果只想先解决当前最扎眼的问题：先做 `Prompt UI-1` 到 `Prompt UI-3`。
- 如果想让招聘者后台整体像专业人才网：做 `Prompt UI-1` 到 `Prompt UI-5`。
- 如果想整站统一交付给领导或客户看：完整做 `Prompt UI-1` 到 `Prompt UI-8`。

## 统一边界

- 本规划只做 UI 精致化，不新增业务能力。
- 不新增后端接口、数据库迁移、权限规则或真实外部能力，除非某轮单独明确。
- 公共简历库、联系解锁、真实下载、真实支付、短信/微信/小程序/App、地图、直播、ATS/RPA 不因 UI 优化而开放。
- 人才服务区仍只展示候选人授权发布快照。
