# LocalTalent Phase 3 Step3：企业招聘工作台生产化

## 开发前必读卡

- **文件定位**：三期第三份 Step 文档；用于把“企业后台演示能力”设计成后续蓝图和 Prompt 可继承的真实企业招聘工作台。
- **输入资料**：`00_PHASE_III_BOUNDARY.md`、`LocalTalent_phase3_step1.md`、`LocalTalent_phase3_step2.md`、`LocalTalent_三期总规划.md`、Phase I/II 企业、职位、投递、面试、导出相关基线。
- **本轮目标**：冻结 company 在三期灰度试运营中必须具备的资料维护、认证材料、职位管理、投递处理、面试邀约和导出申请能力。
- **禁止越界点**：不写代码，不新增迁移，不生成三期蓝图或提示词，不实现公共简历库、受控找人才、联系解锁、会员特权、真实支付或任何外部同步能力。
- **输出边界**：只定义企业招聘工作台的产品范围、状态口径、字段边界、审计要求和验收关注点。
- **下一步影响**：本文件稳定后，下一份建议文档是 `LocalTalent_phase3_step4.md：运营后台生产化与门户运营化`。

**统一硬边界提醒**：企业中心是 company 私有域；企业公开主页与企业中心必须彻底隔离；企业只能访问本企业数据；未认证企业不得上线职位；投递池不是公共简历库；导出仍必须走审批、字段级权限和审计链。

## 1. Step1/Step2 继承结论

Step1 已经冻结三期目标是 **小范围真实试运营**；Step2 已经把 candidate 侧真实闭环收口。Step3 只承接 company 工作台，不处理运营后台、商业化、风险池能力或公开门户运营配置。

| 继承结论 | Step3 继承方式 |
| --- | --- |
| 三期目标是灰度试运营，不是公网正式运营。 | Step3 只要求企业招聘工作台真实可用，暂不追求会员商业化、广告结算或大规模企业服务。 |
| 企业中心是本企业私有域。 | 所有企业资料、职位、投递、面试、导出申请必须服务端强制 company 数据域。 |
| 企业公开主页与企业中心隔离。 | 公开页只展示企业公开白名单；企业中心可管理私有资料和审核材料，但不得公开泄露。 |
| 未认证企业不得上线职位。 | 职位发布、编辑、送审、上线必须依赖企业认证状态和运营审核状态。 |
| 投递池不是公共简历库。 | 企业只能处理投递到本企业职位的 application，不得扩展成搜索简历、找人才或联系解锁。 |
| 导出是高风险动作。 | Step3 只规划导出申请入口和状态查看；下载、生成、字段裁剪继续继承一期审批链。 |

Step3 的关键问题是：真实企业能不能在不越权、不泄露候选人数据的前提下，完成从认证到职位发布、投递处理、面试邀约和导出申请的招聘流程。

## 2. 一二期企业能力基线

一二期已经完成了企业侧基础能力，三期 Step3 不应重复造轮子。

| 能力 | 已有基线 | Step3 增量 |
| --- | --- | --- |
| 企业注册登录 | 已有 company 身份、本地账号/JWT 与登录注册入口。 | 三期后续按总规划接入认证与身份映射；Step3 只定义 company 招聘工作台。 |
| 企业认证 | 已有企业认证三态：待审核、通过、驳回。 | 补企业资料维护、认证材料提交、驳回后补正、认证状态解释。 |
| 职位状态 | 已有职位待审核、在线、下线与门户可见性。 | 补职位创建、编辑、预览、送审、下线、驳回修改再提交。 |
| 门户职位可见性 | 公开职位只展示在线且审核通过、企业已认证的职位。 | 继续继承，不允许前端或企业中心绕过审核直接上线。 |
| 企业公开页 | 二期已有找企业与企业公开主页。 | 企业中心维护公开资料后，公开页只读取公开白名单字段。 |
| 投递池 | 已有企业投递池仅查本企业 applications。 | 补筛选、详情、备注、阶段流转、面试邀约入口。 |
| 面试场次 | 已有面试场次创建、签到码和签到记录。 | 补企业工作台里的邀约与场次状态管理口径。 |
| 导出申请 | 已有导出审批与一次性下载链路。 | 企业侧只提供申请入口和状态查看，不直接下载未审批数据。 |

Step3 允许企业中心处理本企业私有资料、认证材料和投递信息，但所有公开展示、导出、审计、字段访问都必须沿用一二期边界。

## 3. Step3 总目标

三期企业招聘工作台的目标是：

```text
company 能完成：维护企业资料 → 提交认证材料 → 创建/编辑职位 → 提交审核 → 审核通过后公开可见 → 处理投递 → 邀约面试 → 申请导出。
```

达到灰度试运营口径时，company 至少应能做到：

1. 登录后进入企业中心，看到企业认证状态、职位统计、待处理投递、面试邀约、导出申请状态。
2. 维护本企业资料，并清楚区分公开资料、认证材料和后台私有资料。
3. 提交认证材料，查看审核进度、驳回原因并补正。
4. 创建、编辑、预览职位，提交审核，审核通过后才在门户可见。
5. 处理本企业职位投递，筛选 application，查看投递详情，记录备注和阶段流转。
6. 基于本企业 application 发起面试邀约，查看面试和签到状态。
7. 发起导出申请，查看审批、生成、下载链接发放状态，但不得绕过审批直接导出。

## 4. 企业中心信息架构

Step3 冻结企业中心的信息架构，不直接展开成接口或数据库 schema。

| 模块 | 目标能力 | 状态与边界 |
| --- | --- | --- |
| 企业首页 | 展示认证状态、在招职位、待审职位、待处理投递、面试邀约、导出申请摘要。 | 只展示本企业数据；状态来自服务端。 |
| 企业资料 | 维护企业名称、行业、规模、性质、城市、公开简介、Logo 占位等。 | 公开字段和私有字段分层；公开页只读白名单。 |
| 认证材料 | 提交或补正营业执照、联系人、认证说明等材料。 | 认证材料不得进入公开企业主页；运营审核后才改变 auth_status。 |
| 职位管理 | 创建、编辑、预览、提交审核、下线职位。 | 未认证企业不得上线职位；状态由服务端返回。 |
| 投递池 | 按职位、状态、时间筛选本企业 applications。 | 不是公共简历库；不得跨企业访问。 |
| 投递详情 | 查看投递摘要、候选人授权字段、流程状态、企业备注。 | 联系方式和高敏字段必须受字段级权限与导出审批约束。 |
| 阶段流转 | 查看/更新投递阶段，例如待处理、已查看、邀约面试、已拒绝。 | 写动作需要 trace、审计和对象级授权。 |
| 面试邀约 | 基于本企业 application 创建邀约或场次，查看签到状态。 | 不接真实短信/微信/App；通知默认站内。 |
| 导出申请 | 提交导出申请、查看审批与生成状态、跳转已授权下载入口。 | 仍走 `export_apply`，未审批不显示下载入口。 |

企业中心可以像招聘 CRM，但不能把“本企业投递处理”扩展成“平台全量找人才”。

## 5. 状态机与服务端口径

Step3 的核心纪律是：**企业侧所有状态以服务端返回为准**。

| 状态域 | 服务端应返回的状态口径 | 前端禁止行为 |
| --- | --- | --- |
| company | 待认证、认证通过、认证驳回、资料待补正、禁用或冻结。 | 禁止前端根据资料完整度推断认证通过。 |
| auth | 未提交、待审核、通过、驳回、补正中。 | 禁止跳过运营审核改变认证结果。 |
| job | 草稿、待审核、审核通过、审核驳回、在线、下线。 | 禁止未认证企业或未审核职位公开可见。 |
| application | 新投递、已查看、邀约面试、已签到、已拒绝、已归档。 | 禁止前端按按钮点击自行构造阶段。 |
| interview | 待邀约、已邀约、已确认、已签到、已取消、已过期。 | 禁止外部短信/微信/App 作为默认通知渠道。 |
| export | 待审批、通过、驳回、生成中、已生成、生成失败、链接已发放、已过期。 | 禁止未审批展示下载入口。 |

任何写动作成功后，页面应重新读取服务端状态。企业中心的体验可以乐观提示“处理中”，但最终业务状态必须来自服务端。

## 6. 字段与权限边界

### 6.1 本企业私有域

company 在企业中心内，可以维护和查看本企业必要字段，例如企业资料、认证材料、职位草稿、投递详情、企业备注、面试邀约和导出申请。

要求：

- 必须 company 登录。
- 必须强制当前 company 只能访问本企业数据。
- 所有含对象 ID 的企业、职位、投递、面试、导出接口必须覆盖 IDOR/BOLA。
- 非 company、未登录、跨 company 访问必须返回统一无权限响应与 trace。

### 6.2 企业公开域

公开企业列表和企业主页只能展示公开白名单字段。

允许公开的典型字段：

- 企业名称、Logo 或占位标识。
- 城市、行业、性质、规模、认证标签。
- 公开简介摘要。
- 在招职位数和在线职位列表。
- 更新时间。

禁止公开的字段和内容：

- 联系人手机号、邮箱、内部电话。
- 营业执照附件、认证材料、审核材料。
- 后台备注、风控结论、运营处理记录。
- 导出记录、投递池、审计记录。

### 6.3 投递池候选人字段

企业投递池只能展示与本企业职位投递相关的候选人摘要和流程字段。

要求：

- 不得展示未授权联系方式。
- 不得展示简历正文、附件对象 key、证据或原始 JSON。
- 高敏字段必须通过字段级权限裁剪或脱敏。
- 企业备注只能属于本企业私有域，不得进入公开门户或导出旁路。

### 6.4 导出字段

导出仍是高风险动作。

要求：

- 必须先提交 `export_apply`。
- 必须 operator 审批通过。
- 必须异步生成。
- 必须字段级权限裁剪。
- 必须写 `audit_log` 与 `field_access_log`。
- 未审批、驳回、未生成、链接已发放或过期时，不得展示下载入口。

## 7. 审计、幂等与数据域

Step3 后续实现时，以下动作必须具备 trace、幂等、防重复或审计口径。

| 动作 | 要求 |
| --- | --- |
| 企业资料保存 | 防重复提交；记录更新时间和 trace；区分公开字段与私有字段。 |
| 认证材料提交 | 写 `audit_log`；状态进入待审核；驳回原因由运营返回。 |
| 职位创建/编辑 | 幂等或防重复提交；未认证企业不得提交上线；写 audit_log。 |
| 职位送审/下线 | 写 audit_log；上线必须依赖运营审核通过。 |
| 投递阶段流转 | 仅限本企业 application；写 audit_log；覆盖 IDOR/BOLA。 |
| 企业备注 | 仅本企业可见；不得公开或导出旁路泄露。 |
| 面试邀约 | 基于本企业 application；默认站内通知；写 audit_log。 |
| 导出申请 | 继承一期导出审批；申请、审批、生成、下载链接发放均可追踪。 |

数据域必须覆盖：

1. 企业资料只能本企业访问。
2. 职位只能本企业创建、编辑、送审、下线。
3. 投递池只能查看本企业职位产生的 application。
4. 面试邀约只能基于本企业 application。
5. 导出申请只能由本企业创建并查看本企业申请。
6. 运营审核和审计另由后续 Step 设计，不由企业中心绕行。

## 8. 自动化验收关注点

Step3 后续拆 Prompt 时，每轮至少需要覆盖以下验收点。

| 验收点 | 关注内容 |
| --- | --- |
| 跨企业 IDOR/BOLA | 企业 A 不能访问或操作企业 B 的资料、职位、投递、面试、导出申请。 |
| 未认证企业阻断 | 未认证企业不得上线职位；门户不展示未认证企业职位。 |
| 职位可见性 | 只有企业认证通过、职位审核通过且在线的职位可公开可见。 |
| 投递池字段裁剪 | 不展示未授权手机号、邮箱、简历正文、附件、证据、原始 JSON。 |
| 投递阶段审计 | 阶段流转、拒绝、邀约等动作有 audit_log 和 trace。 |
| 面试邀约边界 | 只能基于本企业 application；不接真实短信/微信/App。 |
| 导出审批 | 未审批不可下载；字段级权限和审计链继续生效。 |
| auditor/operator 分权继承 | 企业中心不授予 auditor 写能力；运营写动作留给运营后台。 |

建议后续实现阶段保留或新增以下测试方向：

- `CompanyProfileCertificationIT`：企业资料、认证材料、驳回补正、跨企业阻断。
- `CompanyJobManagementIT`：职位创建、编辑、送审、下线、未认证阻断。
- `CompanyApplicationWorkbenchIT`：投递池筛选、详情、阶段流转、字段黑名单。
- `CompanyInterviewInvitationIT`：面试邀约、站内通知、跨企业 application 阻断。
- `CompanyExportEntryIT`：导出申请入口、状态查看、未审批下载隐藏。
- 前端 company 工作台状态测试：loading、empty、error、unauthorized、pending_review、rejected、approved。

这些测试名称只是规划建议，不是本轮要创建的代码文件。

## 9. Step3 结论与下一步

Step3 结论：

- 三期企业工作台应优先让 company 真实完成招聘流程，而不是扩展公共简历库或商业化能力。
- 企业中心是本企业私有域；公开企业主页只展示公开白名单字段。
- 未认证企业不得上线职位，未审核职位不得公开可见。
- 投递池只处理本企业职位产生的投递，不是找人才、搜索简历或联系解锁入口。
- 面试邀约默认走站内通知，不接真实短信、真实微信、小程序或 App。
- 导出只规划申请入口与状态查看，必须继承一期审批、字段裁剪、MinIO 短期下载和审计链。
- 所有对象 ID 必须覆盖 IDOR/BOLA。

建议下一份文档：

```text
LocalTalent_phase3_step4.md：运营后台生产化与门户运营化
```

Step4 应聚焦运营侧，不建议直接跳到商业化或风险池能力：

1. 企业审核生产化。
2. 职位审核生产化。
3. 内容与活动管理生产化。
4. 首页与频道推荐位配置。
5. 风险审核。
6. 审计中心 UI。
7. 公开门户真实状态与后台配置驱动。
