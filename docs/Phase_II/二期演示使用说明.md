# LocalTalent 二期演示使用说明

本文面向第一次接手 LocalTalent 的新同事。照着做，你可以在本地看到二期高保真门户：`首页 / 找工作 / 名企直聘 / 急聘 / 今日招聘 / 找企业 / 人才服务区 / 招聘会 / 网络招聘会 / 校园招聘 / 就业政策 / 职场资讯 / HR 工具箱 / 帮助中心 / 更多服务 / 登录注册入口`。

二期不是重做一期底座，而是在一期合规、安全、审计与 Next/SSR 基础上，把公开门户做得更像一个完整地方人才网。

## 一、演示边界

- 二期演示数据只用于本地开发和验收，不是生产初始化数据。
- 演示数据通过 `scripts/seed_phase2_demo_data` 显式导入，不进入 Flyway，不影响 CI 空库迁移。
- 人才服务区仍只展示 `candidate_publish_snapshot` 发布快照白名单字段，不展示手机号、邮箱、简历正文、附件、证据或原始 JSON。
- 公开门户不做公共简历库，不提供“搜索简历 / 查看简历 / 联系解锁 / 受控找人才”能力。
- 对接接口仍为 stub，不连接 ATS/RPA、受控候选池或外部主库。
- 地图、视频、直播、短信、微信、小程序、App、支付、会员商业化均为占位或风险池，不接真实外部能力。
- 企业公开页只展示公开字段和在线职位，不展示联系人、营业执照附件、审核材料、后台备注、投递池、导出记录或审计记录。

## 二、你现在可以看到什么

- 高保真门户首页：顶部工具条、Logo 搜索区、主导航、职位分类墙、快捷入口、登录卡、公告/招聘会 tab、广告位、扫码 CTA、推荐企业、热招职位。
- 找工作：公开职位搜索、筛选 query、职位卡片、名企直聘入口、职位详情 SEO 入口、投递/收藏占位。
- 找企业：认证企业列表、企业公开主页、公开简介、认证标签、在招职位聚合。
- 人才服务区：参考 demo 简历库布局，但只展示发布快照卡片、低风险筛选、分页和风险提示。
- 招聘会与内容频道：招聘会列表/详情、就业政策、职场资讯、公告、HR 工具箱、帮助中心。
- 更多服务：今日招聘、快速招聘、校园招聘、网络招聘会等低风险入口；地图、视频、直播、自由职业、求职登记等高风险入口为禁用或风险池说明。
- P1 公开频道补充：`/jobs/famous`、`/jobs/emergency`、`/daily-jobs`、`/job-fairs/online`、`/job-fairs/campus`、`/campus`、`/campus/jobs`、`/campus/elections`、`/explain`、`/explain/[id]` 均为稳定公开入口。
- 登录注册入口：求职者/招聘者登录注册 UI；短信、微信、小程序、App 登录均为禁用占位。
- 一期能力仍可演示：求职者中心、企业后台、运营后台、导出审批、审计 trace、对接 stub。

## 三、本地环境检查

建议版本：

- Docker Desktop 或 Docker Engine
- JDK 21
- Node 24 LTS
- npm

检查命令：

```bash
docker --version
java -version
node -v
npm -v
```

如果 Docker 不可用，先打开 Docker Desktop。后端测试、本地 MySQL、Redis 和 MinIO 都依赖 Docker。

## 四、启动依赖服务

在仓库根目录执行：

```bash
docker compose -f infra/docker-compose.yml up -d
```

这会启动三类本地依赖：

- MySQL：业务数据库，默认容器名 `localtalent-mysql`。
- Redis：本地缓存占位，默认容器名 `localtalent-redis`。
- MinIO：本地对象存储，用于一期导出文件与短期预签名下载演示，端口 `9000/9001`。

健康检查：

```bash
docker exec localtalent-redis redis-cli ping
docker exec -e MYSQL_PWD=localtalent localtalent-mysql mysql -ulocaltalent localtalent -e "SELECT 1;"
curl http://127.0.0.1:9000/minio/health/live
```

## 五、启动后端

打开一个终端：

```bash
cd backend
./mvnw spring-boot:run
```

第一次启动时，Flyway 会自动创建表并执行基础 seed。看到应用启动完成后，在另一个终端验证：

```bash
curl http://127.0.0.1:8080/health
```

也可以用打包方式启动：

```bash
cd backend
./mvnw clean package
java -jar target/backend-0.1.0-SNAPSHOT.jar
```

## 六、导入二期演示数据

后端至少启动过一次并完成 Flyway 迁移后，在仓库根目录执行：

```bash
./scripts/seed_phase2_demo_data
```

脚本会做这些事：

- 检查 `localtalent-mysql` 容器是否在运行。
- 检查核心表和二期字段是否存在，避免 Flyway 尚未迁移时导入。
- 导入 `demo/localtalent_phase2_demo_data.sql`。
- 校验发布快照 JSON 不包含 `mobile/email/resume_body/attachment_object_key/evidence/password_hash/base_profile_json/education_json/experience_json/skills_json`。
- 输出 `PASS`、二期演示账号、核心页面 URL、公开职位/企业/招聘会/内容数量和安全校验结果。

如果要恢复一个干净演示环境：

```bash
docker compose -f infra/docker-compose.yml down -v
docker compose -f infra/docker-compose.yml up -d
cd backend && ./mvnw spring-boot:run
./scripts/seed_phase2_demo_data
```

`down -v` 会删除本地 Docker volume，请只在开发演示环境使用。

## 七、启动前端

打开另一个终端：

```bash
cd frontend
npm ci
npm run dev
```

默认访问：

- 前端：`http://localhost:3000`
- 后端：`http://localhost:8080`

Next 服务端渲染默认通过 `LOCALTALENT_API_BASE_URL=http://localhost:8080` 访问后端。二期公开页面使用 Next/SSR，方便 SEO 和首屏展示。

## 九、登录方式

二期已经有 `/auth/login` 和 `/auth/register` 页面，可以直接用浏览器登录：

```text
http://localhost:3000/auth/login
http://localhost:3000/auth/register
```

也可以用登录助手获取 token 并写入浏览器 `localStorage`。示例：二期企业账号。

```bash
./scripts/demo_login_tokens p2-company
```

脚本会输出类似下面的浏览器 Console 命令：

```js
localStorage.setItem('localtalent_access_token', '...');
localStorage.removeItem('localtalent_admin_role_hint');
location.href = '/company';
```

打开浏览器 DevTools Console，粘贴执行即可进入对应页面。

常用登录助手：

```bash
./scripts/demo_login_tokens p2-candidate
./scripts/demo_login_tokens p2-revoked
./scripts/demo_login_tokens p2-pending
./scripts/demo_login_tokens p2-company
./scripts/demo_login_tokens operator
./scripts/demo_login_tokens auditor
```

退出登录：

```js
localStorage.removeItem('localtalent_access_token');
localStorage.removeItem('localtalent_admin_role_hint');
```

## 十一、MinIO 和导出怎么看

二期公开门户本身不新增导出能力，但一期导出审批仍可用。MinIO 用于本地导出文件存储和短期预签名下载链接。

MinIO 控制台：

```text
http://localhost:9001
```

默认本地账号密码以 `infra/docker-compose.yml` 为准。导出文件所在 bucket 通常为 `export`。

演示边界：

- 未审批导出不可下载。
- 审批通过后后端异步生成 CSV。
- 下载 URL 是短期预签名链接。
- 导出字段仍受字段级权限控制。
- 导出申请、审批、生成和下载发放都要有审计记录。

## 十三、常见问题

### `seed_phase2_demo_data` 提示核心表缺失

先启动后端，让 Flyway 完成迁移：

```bash
cd backend
./mvnw spring-boot:run
```

然后重新执行：

```bash
./scripts/seed_phase2_demo_data
```

### `seed_phase2_demo_data` 提示 V0012/V0013 字段缺失

说明数据库不是当前二期版本。建议在本地演示环境重建 volume：

```bash
docker compose -f infra/docker-compose.yml down -v
docker compose -f infra/docker-compose.yml up -d
cd backend && ./mvnw spring-boot:run
./scripts/seed_phase2_demo_data
```

### 页面列表为空

确认三件事：

- 后端仍在运行。
- 已执行 `./scripts/seed_phase2_demo_data`。
- 前端服务端渲染能访问 `http://localhost:8080`。

### 登录失败

确认使用默认密码：

```text
LocalTalent@123456
```

如果仍失败，查看响应体和页面上的 `trace_id`，再按 trace 查后端日志。

### 前端页面显示无权限

浏览器 Console 清空旧 token 后重新登录：

```js
localStorage.removeItem('localtalent_access_token');
localStorage.removeItem('localtalent_admin_role_hint');
```

再访问 `/auth/login` 或使用：

```bash
./scripts/demo_login_tokens p2-company
```

### 人才服务区没有数据

确认筛选条件：

```text
city_code=310000
category_code=software
```

再重新导入演示数据：

```bash
./scripts/seed_phase2_demo_data
```

### `npm run build` 后 `frontend/next-env.d.ts` 变更

这是 Next 构建生成噪音。若它不是你本轮需要提交的内容，只恢复该文件，不做功能改动。

### MinIO 控制台进不去

先确认容器还在：

```bash
docker compose -f infra/docker-compose.yml ps
```

再确认健康检查：

```bash
curl http://127.0.0.1:9000/minio/health/live
```

## 十四、当前限制

- 二期公开门户是高保真结构和本地演示数据，不是生产上线方案。
- 二期演示数据不是生产 seed，不进入 Flyway。
- 首页、职位、企业、招聘会、资讯等部分演示内容为本地公开演示数据。
- 地图找工作、视频招聘、直播招聘、真实短信、真实微信、小程序、App、真实支付、会员商业化、自由职业交易仍是占位或风险池。
- 人才服务区不是公共简历库，不提供联系方式、不提供查看原始简历、不提供联系解锁。
- 受控找人才、联系解锁、会员商业化、真实 ATS/RPA/受控候选池同步必须先走二期 Step6 风险准入规则，不能直接开发。
- 二期未做像素级截图比对，验收以 DOM 结构、字段黑名单、SEO 构建、脚本闸门和 Runbook 可操作性为准。

## 十五、一句话理解二期

二期把一期的合规底座包装成更像完整地方人才网的公开门户：页面更丰富、入口更完整、搜索和频道更像 demo，但隐私边界仍不变，人才服务区只展示发布快照，高风险能力只占位或进入风险池。
