# LocalTalent Phase II Demo Inventory

## 开发前必读卡

- **文件定位**：二期 demo 页面盘点与高保真门户参考清单；用于支撑二期蓝图和 Prompt 15-23 重排。
- **实现约束**：本文件只描述目标页面结构，不代表立即开发；所有实现仍必须逐轮执行当前 Prompt。
- **禁止越界点**：demo 中的简历库、搜索简历、地图、视频、直播、短信、微信、小程序、App、会员商业化、真实支付、ATS/RPA/受控候选池同步不得直接照搬。
- **下游影响**：Prompt 15 负责同步与闸门；Prompt 16-23 按本盘点逐步落地页面结构、字段白名单、SEO 与验收。
- **自动化验收关注点**：人才服务区只展示发布快照；公开门户不输出原始候选人数据；DOM 不渲染黑名单字段；风险能力只占位或进入风险池。

**统一硬边界提醒**：人才服务区仅展示发布快照；对接接口仅 stub；不得输出原始候选人数据；同意、撤回、数据域、字段级权限必须在服务端收口。

## 5. 高风险改写规则

- **简历库**：不得照搬为公共简历库；公开侧统一改写为人才服务区，唯一数据源为发布快照。
- **搜索简历/找人才**：不得给游客开放；企业侧未来必须走企业认证、权限、字段级权限、候选人授权、审计和联系解锁规则。
- **联系方式**：公开门户、人才服务区、列表页和首页不得展示手机号、邮箱、微信、聊天记录和未授权联系方式。
- **地图/视频/直播/短信/微信/小程序/App**：可以保留入口视觉，但必须明确禁用或占位，不接真实外部服务。
- **会员商业化/广告售卖/展位售卖/支付**：可以做运营展示位或静态占位，不接真实支付、套餐权益或结算。
- **求职登记/快速招聘/自由职业**：涉及个人信息采集或交易撮合，必须进入风险池或后续专项。

## 7. 自动化验收建议

- 首页 DOM 必须包含：顶部工具条、Logo 搜索区、主导航、职位分类墙、快捷入口、登录卡片、公告 tab、广告位、明星企业/推荐职位区、浮动二维码占位。
- 公开字段黑名单必须覆盖：手机号、邮箱、简历正文、附件对象 key、证据、密码哈希、原始候选人 JSON、审核材料、聊天记录。
- 人才服务区必须继续通过 `scripts/check_portal_snapshot_fields`。
- 所有公开列表页必须有 SSR/SEO 构建测试和字段黑名单测试。
- 所有私有入口只做导航体验，真实权限必须由后端 RBAC、数据域和字段级权限强制。
