# LocalTalent Phase-2 Step 4：求职者中心与企业中心能力细化

## 开发前必读卡

- **文件定位**：二期第四份规划文档，只冻结求职者中心与企业中心的信息架构、页面状态、字段边界和验收关注点。
- **实现约束**：本文件是规划输入，不是开发 Prompt；不新增代码、不新增接口 schema、不新增数据库迁移、不生成二期蓝图或提示词。
- **禁止越界点**：禁止受控找人才、联系解锁、会员商业化、真实支付、真实短信、真实微信、小程序、App、真实地图、视频、直播、真实 ATS/RPA 或受控候选池同步。
- **下游影响**：为后续求职者中心、企业中心、运营后台和二期蓝图拆分提供页面范围、状态口径和权限边界。
- **自动化验收关注点**：后续实现必须继续守住人才服务区、发布快照、原始候选人数据、同意、撤回、对接接口、数据域、字段级权限。

**统一硬边界提醒**：人才服务区仅展示发布快照；对接接口仅 stub；不得输出原始候选人数据；同意、撤回、数据域、字段级权限必须在服务端收口。

## 2. Step1-Step3 继承结论

Step4 继承以下前置结论：

- 求职者中心是 candidate 本人私有域，不是公开门户的一部分。
- 企业中心是 company 本企业私有域，不是公共简历库或人才库入口。
- 管理中心只是身份分流入口，前端守卫只做体验；关键权限仍由后端 RBAC、数据域、字段级权限强制。
- 所有状态必须由服务端返回，前端不得推断同意、撤回、投递、企业认证、职位审核、导出审批状态。
- 所有含对象 ID 的页面和操作都必须覆盖 IDOR/BOLA 阻断。

## 3. Step4 总目标

Step4 的目标是把“转化到中心之后用户能做什么”讲清楚。

| 中心 | 目标 | 本文件冻结内容 |
| --- | --- | --- |
| 求职者中心 | 让求职者管理本人招聘流程与发布状态。 | overview、简历、投递、面试/签到、收藏/订阅、站内通知、同意/撤回。 |
| 企业中心 | 让企业管理本企业招聘流程。 | overview、企业资料与认证、职位管理、投递池、面试邀约、导出入口。 |

Step4 不定义最终 API schema，不新增数据库表，不新增代码实现，只冻结中心页面范围、状态口径和验收边界。

## 4. 求职者中心能力细化

### 4.1 中心首页 overview

求职者中心首页应聚合本人状态：

- 简历完成度。
- 简历更新时间。
- 最近投递职位。
- 投递状态统计。
- 最近面试/签到状态。
- 收藏职位数量。
- 搜索/职位订阅数量。
- 站内通知未读数。
- 同意/撤回与发布状态。

状态来源必须完全来自服务端，不允许前端根据字段存在与否推断 `consent_status`、`publishable_flag` 或发布快照可见性。

### 4.2 简历编辑与预览

二期可以规划求职者简历编辑与预览，但必须保持私有域边界。

允许规划：

- 基础资料编辑。
- 教育经历。
- 工作经历。
- 技能标签。
- 求职意向。
- 附件上传入口。
- 简历预览。
- 简历完成度提示。

禁止：

- 把简历正文、附件对象 key、证据材料带入公开门户。
- 未经同意直接生成公开发布快照。
- 前端自行判断哪些字段可公开。
- 把简历预览页收录到搜索引擎。

### 4.3 投递记录

投递记录展示本人投递流程：

- 投递职位。
- 投递企业。
- 投递时间。
- 当前状态。
- 面试邀约入口。
- 签到结果摘要。

边界要求：

- candidate 只能查看本人投递。
- 投递状态由服务端返回。
- 已撤回同意不等于删除投递历史，但不得继续把候选人展示为可公开联系对象。
- 投递详情不得展示企业内部审核信息。

### 4.4 面试/签到状态

面试/签到模块承接一期投递与面试签到能力：

- 展示待面试、已签到、已过期、已取消等状态。
- 展示面试时间、地点、企业公开名称和职位标题。
- 签到结果只作为流程状态，不等同于同意或授权。
- 签到码输入、二维码扫描等真实交互可后续实现，本 Step 只冻结页面能力。

### 4.5 职位收藏

收藏能力属于本人私有数据：

- 收藏职位列表。
- 收藏时间。
- 职位当前公开状态。
- 职位下线提示。
- 取消收藏。

后续实现要求：

- 收藏与取消收藏必须幂等。
- 收藏不能绕过职位公开可见性。
- 收藏列表默认 noindex。

### 4.6 搜索/职位订阅

订阅能力用于承接公开搜索结果：

- 保存搜索条件。
- 订阅职位关键词。
- 订阅城市、职类、薪资、经验等筛选组合。
- 展示订阅状态和更新时间。
- 取消订阅。

边界要求：

- 订阅属于本人私有数据。
- 真实短信、微信、小程序、App 推送不接入；二期第一阶段只规划站内通知。
- 订阅写操作后续必须幂等并可审计。

### 4.7 站内通知入口

站内通知只规划站内能力：

- 投递状态通知。
- 面试邀约通知。
- 签到提醒。
- 企业反馈通知。
- 同意/撤回状态提示。
- 系统公告。

禁止在 Step4 展开真实短信、微信、公众号、小程序或 App 推送。

### 4.8 同意/撤回与发布状态

求职者中心必须清楚展示：

- 当前是否已同意。
- 是否已撤回。
- 是否可发布。
- 发布快照是否公开可见。
- 撤回入口。
- 撤回后的下线提示。

必须遵守：

- 状态由服务端返回。
- 撤回后人才服务区不可见。
- 不在前端推断发布状态。
- 不展示同意证据明文。

## 5. 企业中心能力细化

### 5.1 企业首页 overview

企业中心首页应聚合本企业招聘状态：

- 企业认证状态。
- 在招职位数。
- 待审核职位数。
- 下线职位数。
- 投递池待处理数量。
- 面试邀约数量。
- 导出申请状态摘要。
- 审核驳回提示。

所有统计必须限定本企业数据域。

### 5.2 企业资料与认证

企业资料与认证模块承接一期企业认证状态机：

- 未认证。
- 待审核。
- 已通过。
- 已驳回。

页面能力：

- 企业公开资料编辑。
- 企业认证资料提交。
- 驳回原因展示。
- 重新提交入口。
- 认证通过后展示可发布职位入口。

禁止：

- 未认证企业上线职位。
- 公开展示营业执照附件或审核材料。
- 前端绕过审核状态直接展示“已通过”。

### 5.3 职位管理

职位管理模块覆盖本企业职位：

- 职位列表。
- 职位创建。
- 职位编辑。
- 提交审核。
- 审核状态展示。
- 上线状态展示。
- 下线操作。

状态要求：

- 企业只能创建/编辑本企业职位。
- 企业不能直接把职位设为在线。
- 职位公开可见必须满足企业认证通过、职位审核通过、职位状态在线。
- 上线、下线、提交审核等写动作必须审计。

### 5.4 投递池

投递池是本企业私有域，不是公共简历库。

允许展示：

- application_id。
- job_id、职位标题。
- 投递时间。
- 投递状态。
- 候选人发布快照摘要或经过字段级权限裁剪的最小流程摘要。
- 面试邀约状态。
- 签到状态。

禁止展示：

- 非本企业投递。
- 未授权手机号、邮箱。
- 简历正文。
- 附件对象 key。
- 同意证据。
- 候选人原始 JSON。
- 通过前端隐藏替代服务端裁剪。

### 5.5 投递详情

投递详情应服务招聘处理：

- 查看投递基础信息。
- 查看职位和候选人摘要。
- 查看流程状态。
- 发起面试邀约。
- 记录处理状态。

后续实现时必须：

- 校验本企业数据域。
- 任何敏感字段读取写 `field_access_log`。
- 任何处理状态变化写 `audit_log`。
- 不把投递详情复制到公开门户。

### 5.6 面试邀约

面试邀约承接一期面试场次与签到能力：

- 基于本企业投递创建面试邀约。
- 展示面试时间、地点、状态。
- 展示签到状态。
- 展示过期或取消状态。

禁止：

- 对非本企业投递创建面试。
- 让签到码无限重放。
- 把签到记录当作候选人同意。

### 5.7 导出入口

企业中心可以保留导出申请入口，但必须继承一期导出审批链：

- 企业发起 export_apply。
- operator 审批通过或驳回。
- 审批通过后异步生成文件。
- 文件生成完成后才可获取 MinIO 短期预签名下载链接。
- 未审批、驳回、未生成、已过期或已发放后，不应展示可直接下载入口。

导出必须继续遵守字段级权限、导出审批、`audit_log`、`field_access_log` 和 MinIO 短期下载边界。

## 6. 双中心状态机

| 状态 | 求职者中心表现 | 企业中心表现 |
| --- | --- | --- |
| loading | 加载本人 overview、简历、投递、收藏、订阅。 | 加载本企业 overview、职位、投递池、导出。 |
| empty | 暂无简历、暂无投递、暂无收藏、暂无订阅。 | 暂无认证资料、暂无职位、暂无投递、暂无导出。 |
| unauthorized | 未登录或 token 失效，提示登录。 | 未登录或 token 失效，提示招聘者登录。 |
| forbidden | 非 candidate 访问求职者中心。 | 非 company 访问企业中心，或跨企业访问。 |
| error | 展示 trace_id 和重试入口。 | 展示 trace_id 和重试入口。 |
| retrying | 用户触发重试后展示进行中。 | 用户触发重试后展示进行中。 |
| submitted | 简历/同意/投递等提交后等待服务端状态。 | 企业认证、职位提交、导出申请提交后等待服务端状态。 |
| approved | 同意有效、发布快照可见。 | 企业认证通过、职位审核通过。 |
| rejected | 不适用或显示不可发布原因。 | 企业认证或职位审核驳回，显示原因。 |
| revoked/offline | 已撤回，人才服务区不可见。 | 职位下线，门户不可见。 |

状态规则：

- 所有状态必须由服务端返回。
- 前端只做视觉表达和下一步引导。
- 不允许前端从字段缺失、按钮隐藏或本地缓存推断核心业务状态。

## 7. 字段与权限边界

### 7.1 求职者中心

求职者中心允许展示本人私有摘要，但不得泄露给其他身份。

必须禁止：

- candidate A 访问 candidate B 的简历、投递、收藏、订阅、通知。
- company 访问 candidate center。
- operator/auditor 通过前端私有路径读取候选人本人中心。
- 将本人原始候选人数据带入公开门户或人才服务区。

### 7.2 企业中心

企业中心只能访问本企业资料、职位、投递、面试、导出申请。

必须禁止：

- company A 访问 company B 的企业资料、职位、投递池、导出申请。
- 未认证企业上线职位。
- 企业绕过运营审核公开职位。
- 投递池返回未授权联系方式或简历原文。

### 7.3 公开与私有隔离

- 公开门户只展示职位公开字段、企业公开字段、活动/内容公开字段和发布快照白名单字段。
- 私有中心可读取必要流程摘要，但不得进入公开 SEO 页面、公开缓存、公开搜索索引。
- 导出入口必须继承审批与字段级权限，不得绕开一期导出链。

## 8. 审计与验收关注点

后续实现 Step4 相关能力时，至少要覆盖：

- candidate 只能读取本人中心数据。
- company 只能读取本企业中心数据。
- 简历编辑、投递状态、收藏、订阅、通知均不进入公开门户。
- 同意/撤回状态由服务端返回，撤回后人才服务区不可见。
- 企业未认证通过不得上线职位。
- 企业不能跨企业查看投递池、面试邀约或导出申请。
- 投递池不返回未授权手机号、邮箱、简历正文、附件对象 key、证据或原始 JSON。
- 导出入口未审批、未生成、已过期或已发放后不显示直接下载入口。
- 高风险写动作写 `audit_log`，敏感字段读取写 `field_access_log`。

## 9. Step4 结论

Step4 冻结了二期求职者中心与企业中心的能力边界：

- 求职者中心聚焦本人简历、投递、面试/签到、收藏、订阅、通知、同意/撤回状态。
- 企业中心聚焦本企业资料、认证、职位管理、投递池、面试邀约、导出入口。
- 双中心都是私有域，不是公开门户，也不是公共简历库。
- 状态必须服务端返回，权限必须服务端强制，敏感字段必须可审计。

高保真门户优先补充：

- Step4 的双中心能力仍是二期重要目标，但不应早于公共门户 Shell、首页首屏、找工作、找企业、人才服务区和内容频道的高保真落地。
- 双中心入口应从首页登录卡、职位详情投递入口、企业公开主页和管理中心身份分流自然进入。
- 双中心页面不承担 SEO 公开流量，不进入 sitemap，默认 `noindex`。

Step5 不建议立刻展开受控找人才、联系解锁、会员商业化、真实支付、真实短信/微信/小程序/App、真实地图/视频/直播、真实 ATS/RPA 或受控候选池同步。
