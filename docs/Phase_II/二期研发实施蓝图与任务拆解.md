# LocalTalent 二期研发实施蓝图与任务拆解（高保真门户优先版）

## 开发前必读卡

- **文件定位**：二期从规划进入开发前的实施蓝图；承接 Phase II 边界、demo inventory、Step1-Step6，并指导 Prompt 15-23。
- **实现约束**：二期不是重做一期底座，而是在一期 Next/SSR、权限、审计、导出、对接 stub、CI/Runbook 基础上做高保真人才网门户。
- **禁止越界点**：禁止公共简历库；禁止真实 ATS/RPA/受控候选池同步；禁止真实支付、短信、微信、小程序、App、地图、视频、直播接入；禁止把风险池能力直接排入当前开发。
- **下游影响**：决定 `二期研发提示词.md/html` 的 Prompt 顺序、输出文件、验收命令和边界脚本方向。
- **自动化验收关注点**：首页 DOM 结构、公开字段白名单、人才服务区发布快照、撤回下线、RBAC、数据域、字段级权限、IDOR/BOLA、审计 trace、SEO/Next SSR、风险池闸门。

**统一硬边界提醒**：人才服务区仅展示发布快照；对接接口仅 stub；不得输出原始候选人数据；同意、撤回、数据域、字段级权限必须在服务端收口。

## 1. 执行摘要

二期目标从“补功能”调整为“做成像 demo 一样完整的地方人才网门户”。这意味着二期前半段优先做公开门户观感、页面结构、模块密度、搜索和 SEO，再补求职者/企业/运营中心增强。

本蓝图采用 **高保真门户优先**：

- 以 `docs/Phase_II/demo` 的 24 个 HTML 页面为目标效果来源。
- 以 `LocalTalent_phase2_demo_inventory.md/html` 为页面盘点和 Prompt 映射。
- 公开门户先达到“像人才网”的首屏和频道完整感。
- 私有中心和运营后台保留，但不抢在门户视觉和搜索体验之前。
- 高风险能力只做视觉占位或进入风险池，不因“像 demo”而真实接入。

## 5. 任务拆解

### 5.1 公开门户 Shell

- `TopUtilityBar`：总站/切换城市、会员登录、会员注册、手机端、帮助、网站导航。
- `LogoSearchHeader`：Logo、找工作搜索、高级搜索、地图搜索占位、扫码关注占位。
- `PortalMainNav`：首页、找工作、找企业、人才服务区、招聘会、就业政策、更多服务、管理中心。
- `MoreServicesMenu`：今日招聘、快速招聘、校园招聘、网络招聘会、求职登记、地图、视频、自由职业占位。
- `PortalFooter` 与 `FloatingToolBar`：公开底部导航、二维码占位、回到顶部、简历诊断占位。

### 5.2 首页高保真首屏

- 职位分类墙：分类链接到公开找工作筛选。
- 快捷入口：名企直聘、现场招聘会、网络招聘会、普工招聘。
- 登录卡片：求职者/企业 tab、手机号/验证码视觉、密码登录、注册链接；真实验证码禁用。
- 公告/招聘会 tab：公开 CMS 与活动数据。
- 广告位：通栏广告、推荐位、扫码 CTA；不做真实广告售卖。
- 明星企业/推荐企业/热招职位：只展示企业和职位公开字段。

### 5.3 列表与详情

- 找工作和名企直聘：关键词、城市、职类、薪资、经验、学历、更新时间、排序、分页、职位详情。
- 找企业：企业关键词、城市、行业、性质、规模、认证标签、在招职位数。
- 人才服务区：参考简历库布局，字段只允许 `snapshot_id/display_name_masked/city_code/category_code/skills_summary/experience_years/updated_at`。
- 招聘会/内容：招聘会列表/详情、就业政策、资讯、HR 工具箱、帮助中心。

### 5.4 登录注册与管理中心

- 求职者登录/注册、招聘者登录/注册页面参考 demo 结构。
- 本地账号/JWT 作为可用路径；真实短信、微信扫码、小程序、App 显示禁用占位。
- 管理中心按 candidate/company/operator/auditor 身份分流。
- 私有页面默认 `noindex`，权限仍由服务端强制。

### 5.5 测试与文档

- DOM 结构测试：关键模块必须存在。
- 字段黑名单测试：公开响应和 DOM 不含敏感字段。
- SEO/Next SSR：metadata、sitemap、robots、构建通过。
- 边界脚本：继续守住人才服务区发布快照、风险池能力和外部能力禁入。
- Demo 数据：支撑首页、列表、人才服务区、招聘会、资讯和登录入口演示。

## 6. 接口与字段边界

| 领域 | 允许公开 | 禁止公开 |
| --- | --- | --- |
| 职位 | 标题、城市、职类、薪资展示、经验、学历、福利、企业公开摘要、更新时间。 | 联系人手机号/邮箱、审核材料、营业执照附件、内部备注。 |
| 企业 | 企业名、Logo、城市、行业、性质、规模、认证标签、公开简介、在招职位数。 | 联系人、营业执照附件、审核材料、后台备注。 |
| 人才服务区 | 发布快照白名单字段。 | 手机号、邮箱、简历正文、附件、证据、原始 JSON、聊天记录。 |
| 招聘会/内容 | 公开标题、摘要、正文、时间、地点、主办方、封面、状态。 | 报名名单、候选人明细、联系方式、签到证据、内部备注。 |
| 登录注册 | UI、状态、错误提示、trace。 | 真实短信、真实微信扫码、小程序、App 登录。 |

## 7. 测试矩阵与验收标准

| 验收项 | 方式 |
| --- | --- |
| Demo 同步 | `find docs/Phase_II/demo -name '*.html' | wc -l` 应为 24。 |
| 首页结构 | DOM 测试断言顶部工具条、搜索区、分类墙、登录卡、公告 tab、广告位、推荐企业/职位存在。 |
| 字段黑名单 | 后端响应、前端 DOM、脚本扫描均不含敏感字段。 |
| 人才服务区 | 只读发布快照；撤回后不可见；`check_portal_snapshot_fields` PASS。 |
| SEO/SSR | `npm run build`、metadata、sitemap、robots、私有页 noindex。 |
| IDOR/BOLA | 所有对象详情由服务端判断公开可见或数据域。 |
| 审计 | 写动作写 `audit_log`，敏感字段读取写 `field_access_log`。 |
| 风险池 | 地图、视频、直播、短信、微信、小程序、App、支付、ATS/RPA 等仅占位或风险池。 |
