# LocalTalent 二期 Codex 投喂手册（高保真门户优先 · Prompt 15–24）

> 用途：二期开发前必读 Markdown 版；打开对应 HTML 后可逐轮复制 Prompt 15–24。
> 重要：Prompt 15–23 是二期主线；Prompt 24 是进入三期前的低风险公开频道补充，不代表开放风险池能力。

## 开发前必读卡

- **文件定位**：Codex 二期逐轮投喂执行手册，覆盖 Prompt 15–24。
- **实现约束**：每轮只做当前 Prompt；不得跨 Prompt 偷跑；不得修改一期 Prompt 0–14 既定事实。
- **策略变化**：二期从“功能清单型”调整为“demo 页面驱动型”，第一阶段采用高保真门户优先。
- **禁止越界点**：禁止公共简历库；禁止真实 ATS/RPA/受控候选池同步；禁止公开联系方式；禁止真实支付、短信、微信、小程序、App、地图、视频、直播接入；禁止把风险池能力混入 Prompt 15–23。
- **自动化验收关注点**：demo 同步、首页 DOM 结构、公开字段白名单、人才服务区发布快照、SEO/Next SSR、RBAC、数据域、字段级权限、IDOR/BOLA、审计和风险池闸门。

**统一硬边界提醒**：人才服务区仅展示发布快照；对接接口仅 stub；不得输出原始候选人数据；同意、撤回、数据域、字段级权限必须在服务端收口。

## 失败情形处理矩阵

| 现象 | 优先判断 | 动作 |
| --- | --- | --- |
| 首页不像 demo | 缺 DOM 模块或布局层级 | 补 TopUtilityBar、搜索区、分类墙、登录卡、公告 tab、广告位等结构测试。 |
| 字段泄露 | DTO、搜索索引、缓存或 DOM 漏字段 | 服务端白名单裁剪，前端黑名单测试补齐。 |
| 风险能力偷跑 | 为了像 demo 接入真实地图/短信/微信等 | 删除真实接入，保留占位或进入风险池。 |
| SEO/构建失败 | Next 路由、metadata、sitemap、SSR 数据问题 | 最小修复当前页面和测试。 |
| IDOR/BOLA | 对象级授权缺失 | 在 service 层补数据域校验，补跨用户/跨企业测试。 |
