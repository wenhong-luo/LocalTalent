# 74cms 人才网目标效果与 LocalTalent 二期取舍分析

## 开发前必读卡

- **文件定位**：二期规划的目标站分析资料，用于解释 74cms/demo 给 LocalTalent 带来的页面形态、频道结构、转化路径和功能取舍。
- **实现约束**：本文不是开发 Prompt，不直接要求写代码、建表或新增接口；后续每轮开发仍必须以二期 Step、蓝图和提示词为准。
- **禁止越界点**：不得把 74cms 的“简历库/搜索简历/找人才”照搬成公共简历库；不得提前接真实 ATS、受控候选池、RPA、真实支付、真实短信、真实微信、小程序、App、真实地图、视频或直播能力。
- **下游影响**：用于补充 `LocalTalent_phase2_step1` 之后的页面规划、搜索体系、转化闭环、中心能力细化和高风险功能暂缓清单。
- **自动化验收关注点**：人才服务区、发布快照、原始候选人数据、同意、撤回、对接接口、数据域、字段级权限、审计、IDOR/BOLA。

**统一硬边界提醒**：人才服务区仅展示发布快照；对接接口仅 stub；不得输出原始候选人数据；同意、撤回、数据域、字段级权限必须在服务端收口。

```text
哪些能力风险高，必须等边界和合规方案成熟后再评估？
```

- 一期文档是工程事实来源；二期必须承接一期已完成的 Next/SSR、统一响应体、权限、审计、人才服务区、导出审批、对接 stub 和 CI/Runbook。

## 2. 核心结论

LocalTalent 二期应学习 74cms/demo 的三件事：

1. **地方人才网的公开门户组织方式**：首页、找工作、找企业、招聘会、就业政策、HR 工具箱、更多服务和管理中心入口。
2. **求职者和企业的转化路径**：游客从公开职位进入登录、简历、投递、收藏；企业从公开门户进入注册、认证、发布职位、投递池。
3. **运营型人才站的频道感**：名企、快速招聘、招聘会、资讯、工具箱、帮助中心等入口让网站看起来像完整平台。

LocalTalent 二期不应照搬的三件事：

1. **公共简历库**：LocalTalent 公开层只能做人才服务区，且人才服务区只展示发布快照白名单字段。
2. **高风险外部能力直连**：真实 ATS、RPA、受控候选池、短信、微信、小程序、App、支付、地图、视频、直播都需要后续独立风险评估。
3. **先全功能后补边界**：LocalTalent 已经有一期合规底座，二期必须继续“先边界、后功能”。

二期优先级建议：

```text
先做公开门户和招聘闭环体验
再做求职者中心与企业中心增强
再做运营后台、内容活动和风险治理
最后再评估受控找人才、联系解锁、会员商业化和真实外部能力
```

| 频道 | 典型页面/入口 | 对 LocalTalent 的意义 | 二期取舍 |

| 更多服务 | 地图找工作、快速招聘、自由职业、视频、直播 | 丰富感和长尾频道 | 多数先占位，不接真实外部能力。 |
| 登录注册 | 求职者登录、招聘者登录、会员注册 | 转化关键页 | 必做，但登录态策略按 LocalTalent。 |
| 管理中心 | 求职者中心、企业中心、运营后台入口 | 私有域入口 | 必做身份分流，服务端权限强制。 |

## 4. LocalTalent 二期功能分层

### 4.1 二期第一阶段建议做

这些能力低风险、与招聘闭环强相关，适合进入二期第一阶段规划：

| 能力 | 说明 | 依赖边界 |
| --- | --- | --- |
| 公开首页 | 聚合搜索、热门职位、企业推荐、招聘会/资讯入口 | 只读公开字段；SSR/SEO。 |
| 找工作列表/详情 | 关键词、城市、职类、薪资、经验、学历、更新时间、排序 | 只展示在线职位和认证企业。 |
| 找企业列表/企业主页 | 企业公开信息、认证标签、在招职位 | 不展示营业执照附件、审核材料或内部联系人。 |
| 人才服务区增强 | 发布快照列表、筛选、SEO、视觉增强 | 唯一数据源仍为 `candidate_publish_snapshot`。 |
| 招聘会公开展示 | 活动列表、活动详情、状态和主办方 | 报名名单、候选人明细后续再做。 |
| 就业政策/资讯/HR 工具箱 | 公开内容、搜索、分类、SEO | 不接外部下载或个人数据收集。 |
| 求职者中心增强 | 简历、投递、收藏、订阅、通知、同意/撤回状态 | 本人私有域，状态由服务端返回。 |
| 企业中心增强 | 企业资料、认证、职位管理、投递池、面试邀约、导出入口 | 本企业数据域，写操作审计。 |
| 运营后台增强 | 内容活动、审核队列、风险配置、审计入口 | operator 写、auditor 只读。 |

### 4.2 二期后续可做

这些能力有价值，但不应压到第一阶段：

| 能力 | 暂缓原因 | 后续进入条件 |
| --- | --- | --- |
| 网络招聘会 | 活动状态、报名、展位、在线互动复杂 | 活动模型和报名隐私边界稳定后再做。 |
| 校园招聘 | 组织、院校、批次、岗位专题较重 | 招聘会/活动基础能力完成后再做。 |
| 快速招聘/今日招聘 | 多为运营专题 | 职位搜索和推荐规则成熟后再做。 |
| 地图找工作 | 真实地图 SDK、定位、成本和隐私风险 | 明确供应商、费用、隐私说明后再接。 |
| 视频招聘/直播招聘 | 视频存储、审核、直播供应商、版权风险 | 内容审核和供应商治理完成后再评估。 |
| 自由职业 | 类似独立业务线 | 主招聘闭环稳定后再单独立项。 |
| 智能匹配/推荐 | 排序公平性、解释性和数据边界 | 先做规则推荐，不做黑盒 AI 决策。 |

### 4.3 高风险暂缓，日后评估

这些功能不是“永远不做”，而是必须先进入风险池，等规避方案成熟后再决策：

| 高风险能力 | 当前结论 | 未来评估重点 |
| --- | --- | --- |
| 公共简历库 | 不做 | 是否能改为发布快照公开 + 企业受控检索。 |
| 受控找人才 | 不在当前 Step 展开 | 企业认证、候选人授权、字段级权限、联系解锁、审计。 |
| 联系方式解锁 | 不做 | 权益、审批、授权、候选人撤回、访问留痕。 |
| 会员商业化/套餐 | 不做真实交易 | 权益边界、支付合规、发票、退款、消费审计。 |
| 真实支付 | 不接 | 支付供应商、资金清结算、退款、风控。 |
| 真实短信/微信/小程序/App | 不接 | 供应商合同、模板合规、退订、频控、隐私协议。 |
| 真实 ATS/RPA/受控候选池同步 | 不接 | 数据来源合法性、最小字段、同步责任、撤回联动。 |
| 地图/视频/直播真实服务 | 不接 | 供应商、成本、内容审核、隐私与版权。 |

## 6. 公开字段边界

### 6.1 公开职位字段

允许展示：

- 职位 ID、标题、城市、区域、职类。
- 薪资展示、经验、学历、福利标签。
- 企业公开摘要、认证标签。
- 职位状态、更新时间。

禁止展示：

- 企业内部联系人手机号、邮箱、微信。
- 审核材料、营业执照附件。
- 内部备注、导出记录、审计记录。

### 6.2 公开企业字段

允许展示：

- 企业 ID、企业名、Logo、城市、行业、性质、规模。
- 认证标签、公开简介摘要、在招职位数。

禁止展示：

- 营业执照附件、审核材料、驳回记录。
- 企业后台联系人手机号、邮箱。
- 内部导出、投递池、面试安排、财务权益数据。

### 6.3 人才服务区字段

公开人才服务区只允许展示：

- `snapshot_id`
- `display_name_masked`
- `city_code`
- `category_code`
- `skills_summary`
- `experience_years`
- `updated_at`

公开人才服务区禁止展示：

- 手机号、邮箱。
- 简历正文。
- 附件对象 key。
- 同意或撤回证据。
- `base_profile_json`、`education_json`、`experience_json`、`skills_json`。
- 聊天记录、投递详情、面试记录、内部备注。

### 6.4 私有中心字段

求职者中心和企业中心可以读取必要的私有流程数据，但必须满足：

- 只能当前本人或本企业访问。
- 服务端强制 RBAC、数据域、字段级权限。
- 高风险字段读取写 `field_access_log`。
- 写动作写 `audit_log`。
- 不得把私有中心字段带入公开门户、SEO 页面、缓存或搜索索引。

## 8. 二期规划使用规则

后续写 Step、蓝图或 Prompt 时，必须按以下规则使用本文：

1. 看到“目标站有这个功能”，不等于 LocalTalent 当前要做。
2. 低风险且能增强招聘闭环的能力，可以进入二期第一阶段。
3. 高风险但有业务价值的能力，进入“后续评估池”，不直接进入开发 Prompt。
4. 涉及候选人原始数据、联系方式、导出、对接、支付、消息触达的能力，必须先写边界和风控，再进入实现。
5. 所有公开页面必须写字段白名单。
6. 所有私有中心页面必须写数据域和 IDOR/BOLA 阻断测试。
7. 所有敏感字段访问必须能审计。

## 9. 建议进入二期 Step 的内容

已经完成或正在规划的 Step 可以按以下节奏推进：

| Step | 建议主题 | 来源 |
| --- | --- | --- |
| Step1 | 产品定位、门户信息架构、页面分层、边界总纲 | demo 全站结构 + 一期基线 |
| Step2 | 门户公开页面与搜索体系 | 首页、找工作、找企业、招聘会、资讯、HR 工具箱 |
| Step3 | 求职者与企业转化闭环 | 登录注册、投递、收藏、订阅、企业认证、职位发布 |
| Step4 | 求职者中心与企业中心能力细化 | 我的投递、我的收藏、面试邀请、职位管理、投递池 |
| Step5 | 运营后台与内容活动能力细化 | 审核、CMS、活动、资讯、HR 工具箱、风险治理 |
| 后续 Step | 受控找人才、联系解锁、会员商业化、推荐、外部能力 | 必须先风险评估再排期 |

## 10. 本文件结论

74cms/demo 对 LocalTalent 最大的价值，不是它的旧技术栈或某个具体接口，而是它展示了地方人才网应该具备的“完整感”：

- 首页要像门户。
- 找工作和找企业要像搜索产品。
- 人才服务区要像有供给，但不能变成公共简历库。
- 求职者中心和企业中心要承接转化。
- 运营后台要能审核、配置、留痕。
- 更多服务可以先像平台，但高风险能力不能先做实。

LocalTalent 二期的正确做法是：

```text
低风险功能先做体验闭环；
高风险功能先做边界和风险池；
等规避方案成熟后，再决定是否实现。
```
