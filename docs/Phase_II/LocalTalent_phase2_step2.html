2. 高保真门户优先补充

完整 demo 已同步到 docs/Phase_II/demo，并由 LocalTalent_phase2_demo_inventory.md/html 逐页盘点。Step2 的公开门户规划必须从“频道完整”升级为“页面结构高保真”。

  • 首页必须覆盖顶部工具条、Logo 搜索区、主导航、职位分类墙、快捷入口、登录卡片、公告/招聘会 tab、广告位、明星企业/推荐职位和浮动二维码占位。
  • 找工作与名企直聘必须覆盖职位卡片、分类墙、投简历/收藏入口、认证/名企标签和扫码求职 CTA。
  • 找企业必须覆盖企业卡片、行业/性质筛选、在招职位数、企业热度等公开展示结构。
  • 人才服务区参考 demo 简历库布局，但必须只展示发布快照白名单字段。
  • 招聘会、就业政策、HR 工具箱和更多服务按 demo 页面形成频道完整感；地图、视频、直播、自由职业、求职登记等高风险能力只做占位或进入风险池。

3. 公开门户总原则

公开门户面向游客、搜索引擎、未登录用户和公开浏览场景。它要承接 74cms demo 的地方人才网体验，但必须使用 LocalTalent 的合规边界重写高风险能力。

首屏清晰搜索稳定数据克制状态明确服务端兜底

不得把人才服务区做成公共简历库；不得公开手机号、邮箱、简历正文、附件对象 key、证据、聊天记录、审核材料、营业执照附件。

3. 页面规划

页面规划口径
首页SSR 首屏渲染，输出城市、主搜索、导航、职类、推荐职位、推荐企业、招聘会和资讯入口。
找工作只展示在线职位和认证企业，支持关键词、城市、职类、薪资、经验、学历、更新时间、排序、分页。
找企业展示已认证企业公开信息和在招职位聚合，不展示联系人与审核材料。
人才服务区只展示已同意、可发布、未撤回的发布快照；唯一数据源是 candidate_publish_snapshot。
招聘会首批只做公开列表与详情；报名、展位、参会名单后置。
就业政策/资讯/HR 工具箱统一纳入公开内容中心，只展示 CMS 公开字段。
更多服务今日招聘可做轻量入口；地图、视频、直播、自由职业、求职登记先占位。
管理中心只做身份分流入口；真正访问由 JWT、RBAC、数据域和字段级权限服务端校验。

4. 搜索体系

频道公开查询维度边界
全站搜索职位、企业、人才服务区、资讯不提供游客公开“搜索简历”。
找工作keyword、city_code、category_code、salary_range、experience_code、education_code、updated_within_days、sort、page/size只返回在线职位和认证企业。
找企业keyword、city_code、industry_code、company_nature、company_scale、verified、has_online_jobs不返回联系人、营业执照附件、审核材料。
人才服务区keyword、city_code、category_code、experience_years、updated_within_days、sort、page/size只查发布快照，不联表原始候选人表。
招聘会/内容keyword、city_code、event_type、status、date、content_type、category_code不返回候选人报名和签到证据。

5. SEO 与 Next/SSR 策略

  • 首页、找工作、职位详情、找企业、企业主页、人才服务区、招聘会、资讯、HR 工具箱优先 SSR 或 ISR 友好。
  • 登录页、管理中心、后台页面、占位页、写操作结果页默认 noindex。
  • 列表页首屏由服务端渲染，筛选变更可由 CSR 增强，但 URL query 必须同步。
  • sitemap 首批覆盖 /、/jobs、/companies、/portal/talent-service-area、/job-fairs、/articles、/hr-tools。
  • 人才服务区 metadata 不写“简历库”，应写“人才服务区”“发布快照”“本地人才摘要”。

6. 公开接口字段白名单

领域允许禁止
职位职位标题、城市、职类、薪资展示、经验、学历、福利、企业公开摘要、状态、更新时间。联系人手机号/邮箱、审核材料、营业执照附件、内部备注。
企业企业名、Logo、城市、行业、性质、规模、认证标签、公开简介、在招职位数。联系人手机号/邮箱、营业执照号全文、营业执照附件、审核材料。
人才服务区snapshot_id/display_name_masked/city_code/category_code/skills_summary/experience_years/updated_atmobile、email、resume_body、attachment_object_key、evidence、password_hash、原始 JSON。
招聘会/资讯/HR 工具箱公开标题、摘要、时间、地点、主办方、公开正文。参会个人名单、候选人联系方式、报名证据、签到证据。

7. Step2 结论

Step2 冻结了二期公开门户的首批页面、搜索维度、SEO/Next SSR 策略和字段白名单。

当前补充结论：Step2 已被 LocalTalent_phase2_demo_inventory.md 扩展为高保真公开门户页面策略；二期第一开发阶段应优先完成公共门户 Shell、首页高保真首屏、找工作/名企直聘、找企业、人才服务区改写和公开内容频道。
