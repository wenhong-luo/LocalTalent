统一硬边界提醒：人才服务区仅展示发布快照；对接接口仅 stub；不得输出原始候选人数据；同意、撤回、数据域、字段级权限必须在服务端收口。

开发前必读卡

  • 文件定位：冻结运营后台、内容活动、招聘会展示配置、风险审核、审计中心运营视角和角色权限边界。
  • 资料来源：边界文件、两份新 MD、Step1-Step4、本地 demo、一期文档。
  • 禁止越界点：禁止受控找人才、联系解锁、会员商业化、真实支付、真实短信/微信/App、真实地图/视频/直播、真实 ATS/RPA。

1. Step5 总目标

能力域目标冻结内容
运营后台让 operator 管理审核、内容、活动、风险和审计入口。运营 overview、审核队列、内容管理、活动/招聘会管理、风险审核、审计入口。
内容活动支撑公开门户的就业政策、资讯、HR 工具箱、公告和招聘会展示。CMS 字段白名单、软删除/下线、公开展示配置、SEO 口径。
权限审计区分 operator 与 auditor。operator 可写，auditor 只读；所有写动作留痕。

Step5 不定义最终 API schema，不新增数据库表，不新增代码实现，只冻结运营后台页面范围、状态口径和验收边界。

2. 运营后台能力细化

模块能力边界
运营首页 overview待审核企业、待审核职位、待审批导出、待处理内容、待处理招聘会、风险审核、最近审计摘要。只展示运营摘要，不展示候选人联系方式或简历正文。
企业审核企业公开摘要、认证状态、提交时间、驳回原因、审核人和审核时间。不展示营业执照附件、审核材料原文、企业联系人手机号或邮箱。
职位审核职位标题、企业公开名称、城市、职类、薪资展示、审核状态、公开状态、驳回原因。未通过企业认证的职位不得上线；auditor 不得写。
导出审批导出申请、申请企业、业务类型、审批状态、生成状态、下载发放状态、驳回原因。未审批不可下载；不得绕过字段级权限导出。

3. 内容与活动能力细化

能力允许规划禁止
内容管理就业政策、职场资讯、HR 工具箱、帮助中心、公告、公开内容块、SEO 字段。候选人手机号、邮箱、简历正文、附件对象 key、证据、营业执照附件、审核材料。
首页推荐位推荐职位、企业、招聘会、资讯、HR 工具箱、公告栏。引用未审核、已下线、已驳回或已撤回对象；付费置顶和广告售卖。
活动/招聘会管理现场招聘会、网络招聘会占位、校园招聘占位、活动列表、活动详情、主办方、时间地点。活动报名、参会名单、展位售卖、在线互动、直播间、真实地图定位。
招聘会公开展示活动 ID、标题、类型、城市、地点、时间、主办方、公开简介、状态。候选人报名名单、联系方式、签到证据、内部审核备注、非公开附件。

内容和活动删除语义应为软删除或下线，不做物理删除。创建、更新、发布、下线、软删除必须写 audit_log。

4. 风险审核与风险池

风险审核对象包括企业资料风险、职位内容风险、内容发布风险、活动/招聘会风险、导出申请风险、审计异常风险和对接接口异常风险。

受控找人才联系解锁会员商业化真实支付真实短信/微信/App真实地图/视频/直播真实 ATS/RPAAI 原始数据分析

高风险能力只进入风险池，不自动进入开发。风险池条目至少记录风险主题、触发场景、涉及数据、当前结论、未来进入条件和需要的审计与测试。

5. 审计中心运营视角

运营后台应提供审计中心入口，但审计中心保持只读查询属性。可按 trace_id、操作人、角色、业务类型、业务 ID、动作类型、时间范围和 open_api client_code 查询。

响应必须脱敏手机号、邮箱、简历正文、附件对象 key、证据、password、secret、token。operator 与 auditor 可查看审计中心；candidate/company 不能进入运营审计中心。

6. 角色权限边界

角色允许禁止
operator审核企业、审核职位、审批导出、管理内容、管理活动、处理风险、查看审计。绕过字段级权限、绕过导出审批、查看未授权原始候选人数据。
auditor查看审核队列、查看内容活动、查看审计中心、查看风险池摘要。任何写接口；审核、发布、下线、软删除、审批导出、调整推荐位。
company查看本企业业务后台。访问运营后台、审批导出、审核职位、查看其他企业数据。
candidate查看本人求职者中心。访问运营后台、查看企业审核材料、查看审计中心。

前端按钮隐藏只做体验。后续实现必须用服务端权限拒绝 auditor 写操作，并返回统一错误与 trace_id。

7. 运营状态机

loadingemptyunauthorizedforbiddenpending_reviewapprovedrejectedpublishedofflinesoft_deletedrisk_pool

状态必须由服务端返回，前端不得用按钮显示、字段缺失或本地缓存推断审核结果。

8. 自动化验收关注点

  • operator 可执行审核、发布、下线、软删除和导出审批；auditor 写接口返回 AUTHZ_403 并包含 trace_id。
  • 内容和活动删除是软删除或下线，不物理删除。
  • 公开内容、招聘会和活动不返回手机号、邮箱、简历正文、附件对象 key、证据或审核材料。
  • 推荐位不得引用未审核、已下线、已驳回或已撤回对象。
  • 招聘会公开展示不包含报名名单、联系方式、签到证据或内部备注。
  • 审核、发布、下线、软删除、风险处理和导出审批写 audit_log。
  • 敏感字段读取写 field_access_log；审计中心响应脱敏并支持 trace_id 串联。

9. Step5 结论

Step5 冻结了二期运营后台与内容活动能力边界：运营后台聚焦审核、内容、活动、风险和审计入口；内容活动聚焦就业政策、资讯、HR 工具箱、公告、首页推荐位和招聘会公开展示配置。

高保真门户优先补充：运营后台能力继续保留，但二期主线会先把公开门户做成接近 demo 的完整人才网，再增强运营后台。

内容、活动、招聘会和推荐位配置必须服务于首页高保真模块、招聘会频道、资讯/HR 工具箱频道；operator/auditor 权限边界不因前端门户视觉增强而放松。
