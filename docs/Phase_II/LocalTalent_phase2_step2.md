## 4. 公开门户总原则

公开门户面向游客、搜索引擎、未登录用户和已登录用户的公开浏览场景。它的目标是承接 74cms demo 的地方人才网体验，但必须使用 LocalTalent 的合规边界重写高风险能力。

公开门户必须做到：

- 首屏清晰：城市、搜索、找工作、找企业、人才服务区、招聘会、政策资讯入口清楚。
- 搜索稳定：筛选条件必须可 URL 化，方便 SEO、分享、回退和缓存。
- 数据克制：公开层只展示公开字段和发布快照白名单字段。
- 状态明确：空态、加载、错误、无权限、重试中状态沿用一期前端状态机。
- 服务端兜底：敏感字段裁剪必须由后端完成，前端字段白名单只是第二道防线。

公开门户不得做到：

- 不得把人才服务区做成公共简历库。
- 不得公开手机号、邮箱、简历正文、附件对象 key、证据、聊天记录、审核材料、营业执照附件。
- 不得用“后端返回敏感字段，前端隐藏”的方式假装安全。
- 不得把 demo 的“搜索简历/找人才”直接暴露给游客。
- 不得在本阶段接入真实地图、视频、直播、短信、微信、小程序、App 或支付能力。

## 4. 页面规划

### 4.1 首页

首页采用 Next/SSR 首屏渲染，优先输出城市、主搜索框、核心导航、职位分类、推荐职位、推荐企业、招聘会和资讯入口。

首批模块建议：

| 模块 | 首批状态 | 说明 |
| --- | --- | --- |
| 城市与搜索 | 必做 | 支持关键词、频道切换、稳定 query 参数。 |
| 热门职类 | 必做 | 参考 demo 大类/小类结构，先用运营配置或字典。 |
| 推荐职位 | 必做 | 只展示在线职位和认证企业公开摘要。 |
| 推荐企业 | 必做 | 只展示已认证企业公开信息。 |
| 招聘会入口 | 必做 | 展示公开招聘会/活动卡片。 |
| 就业政策/HR 工具箱 | 必做 | 展示公开 CMS 内容。 |
| 移动端/公众号/小程序/App | 占位 | 不接真实外部能力。 |
| 地图搜索/视频招聘/直播招聘 | 占位 | 后续专项，不进入首批实现。 |

### 4.2 找工作

找工作是公开门户最重要的 SEO 列表页，必须支持 SSR 首屏。

首批口径：

- 只展示在线职位，且企业必须认证通过。
- 支持关键词、城市/区域、职类、薪资、经验、学历、更新时间、排序、分页。
- 详情页只展示职位公开字段和企业公开摘要，不展示企业后台联系人手机号、邮箱或审核材料。
- `名企直聘` 首批可以作为认证企业/推荐企业标签进入找工作筛选和首页推荐。
- `紧急招聘/今日招聘` 可以作为运营标签；`智能匹配` 后置到推荐专项。
- `快速招聘` 涉及免注册发布和公开联系方式风险，首批只占位。

### 4.3 找企业

找企业展示已认证企业公开信息和在招职位聚合。

首批口径：

- 支持关键词、城市/区域、行业、企业性质、企业规模、认证标签、是否有在招职位筛选。
- 企业主页展示企业公开简介、认证标识、在招职位列表、招聘会参与情况或资讯扩展。
- 不展示企业联系人手机号、邮箱、营业执照附件、审核材料。
- 地图入口只做占位或后续“地图找企业/地图找工作”专项，不接真实地图 SDK。

### 4.4 人才服务区

demo 中的“简历库/搜索简历/找人才”不能照搬。LocalTalent 公开侧统一改写为人才服务区。

人才服务区公开口径：

- 只展示候选人已同意、可发布、未撤回的发布快照。
- 唯一公开数据源是 `candidate_publish_snapshot`。
- 公开字段只允许 `snapshot_id / display_name_masked / city_code / category_code / skills_summary / experience_years / updated_at`。
- 不允许公开手机号、邮箱、简历正文、附件对象 key、证据、完整教育经历、完整工作经历、完整技能 JSON。
- 候选人撤回同意后，公开列表和详情必须不可见。

企业侧未来的“受控找人才”不属于公开门户，应在后续 Step 独立规划，且必须满足企业认证、权限、字段级权限、权益/审批和审计。

### 4.5 招聘会

首批只做招聘会/活动公开列表与详情。

允许公开展示：

- 活动名称。
- 类型。
- 状态。
- 时间。
- 地点。
- 主办方。
- 公开简介。
- 参会企业数量。
- 公开职位数量。

禁止公开展示：

- 参会个人名单。
- 候选人联系方式。
- 报名证据。
- 签到证据。

网络招聘会、校园招聘、展位费用、在线预定、参会名单后置。

### 4.6 就业政策、资讯与 HR 工具箱

就业政策、职场资讯、HR 工具箱统一纳入公开内容中心。

首批口径：

- 内容来自 CMS 公开字段，支持分类、关键词、发布时间筛选。
- 详情页输出标题、摘要、分类、发布时间、公开正文。
- HR 工具箱下载类内容首批可以展示说明和公开附件占位；如涉及文件下载，必须确保文件是运营公开内容，不是候选人或企业审核材料。

### 4.7 更多服务占位

LocalTalent 二期第一阶段规划为“入口可见，能力克制”：

| 服务 | Step2 口径 | 原因 |
| --- | --- | --- |
| 今日招聘 | 可作为职位筛选标签或列表页快捷入口 | 风险低，复用职位公开字段。 |
| 地图找工作 | 占位 | 需要真实地图 SDK 或地理编码，暂不接。 |
| 快速招聘 | 占位 | demo 形态可能涉及免注册发布和公开联系方式。 |
| 校园招聘 | 占位 | 需要院校、宣讲会、双选会模型，后续独立规划。 |
| 求职登记 | 占位 | demo 需要姓名、联系方式、短信验证码，涉及个人信息收集。 |
| 网络招聘会 | 占位或活动子类型 | 互动、报名、参会名单后置。 |
| 视频招聘 | 占位 | 不接真实视频服务。 |
| 直播招聘 | 占位 | 不接真实直播服务。 |
| 自由职业 | 占位 | demo 有找项目/找人才和服务信息，需单独设计隐私边界。 |
| 帮助中心 | 可做公开内容 | 使用 CMS 公开字段。 |

占位页默认 `noindex`，避免搜索引擎收录未完成能力。

### 4.8 管理中心与公开门户转化入口

首批规划口径：

- 顶部导航保留求职者登录、招聘者登录、会员注册、管理中心入口。
- 未登录用户点击管理中心时进入登录或身份选择，不在公开页面暴露后台数据。
- 已登录用户按身份进入 `/candidate/center`、`/company` 或 `/admin`。
- 登录/注册页面可以规划为二期转化入口，但真实短信、微信扫码、小程序、App 登录不在 Step2 范围。
- 任何企业后台、运营后台、导出审批、审计中心入口都只是导航；真正访问必须继续由 JWT、RBAC、数据域和字段级权限服务端校验。

## 5. 搜索体系

### 5.1 全站搜索

首页全站搜索建议支持四个公开频道：

- `职位`：跳转 `/jobs?keyword=...`。
- `企业`：跳转 `/companies?keyword=...`。
- `人才服务区`：跳转 `/portal/talent-service-area?keyword=...`，只查发布快照摘要。
- `资讯`：跳转 `/articles?keyword=...`。

不提供游客公开“搜索简历”入口。后续企业受控找人才必须另走登录、认证、权限、字段级权限和审计链路。

### 5.2 找工作搜索

找工作公开查询维度冻结为：

- `keyword`：职位标题、企业公开名称、公开标签。
- `city_code`：城市。
- `district_code`：区县，可选。
- `category_code`：职位类别。
- `salary_range`：薪资。
- `experience_code`：经验要求。
- `education_code`：学历要求。
- `company_nature`：企业性质，可选。
- `updated_within_days`：3/7/15/30 天。
- `sort`：默认综合排序，可选最新发布、更新时间、薪资。
- `page / size`：分页。

服务端必须只返回 `status=在线` 且企业认证通过的职位。

### 5.3 找企业搜索

找企业公开查询维度冻结为：

- `keyword`：企业名称、公开简称、公开简介摘要。
- `city_code`：城市。
- `district_code`：区县，可选。
- `industry_code`：行业。
- `company_nature`：企业性质。
- `company_scale`：规模。
- `verified`：是否认证。
- `has_online_jobs`：是否有在线职位。
- `sort`：默认综合排序，可选最新入驻、在招职位数。
- `page / size`：分页。

服务端不得返回企业联系人、营业执照附件、审核材料。

### 5.4 人才服务区搜索

人才服务区公开查询维度冻结为：

- `keyword`：仅匹配发布快照允许字段，如脱敏展示名、技能摘要、职类摘要。
- `city_code`：城市。
- `category_code`：职类。
- `experience_years_min / experience_years_max`：经验年限范围。
- `updated_within_days`：更新时间。
- `sort`：默认最新更新，可选经验年限。
- `page / size`：分页。

服务端必须只查 `candidate_publish_snapshot`，不得在公开接口联表读取 `candidate_user`、`candidate_resume` 等原始候选人数据表。

### 5.5 招聘会与内容搜索

招聘会公开查询维度：

- `keyword`：活动标题、主办方、公开简介。
- `city_code`：城市。
- `event_type`：现场招聘会、网络招聘会、校园招聘占位。
- `status`：未开始、进行中、已结束。
- `date_from / date_to`：时间范围。
- `page / size`：分页。

内容公开查询维度：

- `keyword`：标题、摘要、公开正文。
- `content_type`：就业政策、职场资讯、HR 工具箱、帮助中心。
- `category_code`：分类。
- `published_from / published_to`：发布时间。
- `page / size`：分页。

## 6. SEO 与 Next/SSR 页面策略

二期公开门户应延续一期已选择的 `Next.js + React + Node SSR` 方向，面向几万日活和强 SEO 做页面策略冻结。

### 6.1 SSR/ISR 原则

- 首页、找工作、职位详情、找企业、企业主页、人才服务区、招聘会、资讯、HR 工具箱优先 SSR 或 ISR 友好。
- 登录页、管理中心、后台页面、占位页、写操作结果页默认 `noindex`。
- 列表页首屏由服务端渲染，筛选变更可由 CSR 增强，但 URL query 必须同步。
- 页面必须保留统一加载、空态、错误、无权限、重试中状态。

### 6.2 URL 与 canonical

- 可索引筛选使用稳定 query 参数，不使用临时状态或随机参数。
- 翻页使用 `page` 和 `size`，默认 `page=1`。
- 主要频道保留 canonical，避免不同参数组合造成重复收录。
- 敏感或登录态页面不进入 sitemap。

sitemap 首批建议覆盖：

- `/`
- `/jobs`
- `/companies`
- `/portal/talent-service-area`
- `/job-fairs`
- `/articles`
- `/hr-tools`

robots 策略建议：

- 允许公开门户频道。
- 禁止 `/candidate`、`/company`、`/admin`、`/api`、占位服务页被索引。

### 6.3 Metadata

公开页面 metadata 必须来自公开字段：

- 标题：城市 + 频道 + LocalTalent。
- 描述：首页/列表页使用频道简介；详情页使用公开摘要。
- Open Graph 图片只能使用公开运营图片、企业 Logo 或默认图，不使用候选人原始附件。
- 人才服务区 metadata 不写“简历库”，应写“人才服务区”“发布快照”“本地人才摘要”等合规口径。

## 8. 自动化验收关注点

后续进入代码实现时，Step2 对应的自动化验收至少应覆盖：

- 人才服务区字段黑名单：公开响应和前端 DOM 均不得出现手机号、邮箱、简历正文、附件对象 key、证据、密码哈希、原始 JSON。
- 找工作可见性：未上线职位、未认证企业职位不得公开可见。
- 找企业可见性：未认证企业、审核材料、联系人字段不得公开可见。
- 撤回联动：候选人撤回同意后，人才服务区搜索和列表不可见。
- SEO 构建：公开页面 metadata、sitemap、robots 构建通过。
- IDOR/BOLA：任何详情页对象 ID 都必须由服务端判断公开可见，不得仅靠前端路由隐藏。
- 对接接口：对接接口仍仅 stub，不得因公开门户搜索新增真实 ATS/RPA 或受控候选池同步。
