# LocalTalent Phase-2 Step 5：运营后台与内容活动能力细化

## 开发前必读卡

- **文件定位**：二期第五份规划文档，只冻结运营后台、内容活动、招聘会展示配置、风险审核、审计中心运营视角和角色权限边界。
- **实现约束**：本文件是规划输入，不是开发 Prompt；不新增代码、不新增接口 schema、不新增数据库迁移、不生成二期蓝图或提示词。
- **禁止越界点**：禁止受控找人才、联系解锁、会员商业化、真实支付、真实短信、真实微信、小程序、App、真实地图、视频、直播、真实 ATS/RPA 或受控候选池同步。
- **下游影响**：为后续运营后台增强、内容活动管理、招聘会公开展示、风险治理和二期蓝图拆分提供范围边界。
- **自动化验收关注点**：后续实现必须继续守住人才服务区、发布快照、原始候选人数据、同意、撤回、对接接口、数据域、字段级权限。

**统一硬边界提醒**：人才服务区仅展示发布快照；对接接口仅 stub；不得输出原始候选人数据；同意、撤回、数据域、字段级权限必须在服务端收口。

## 2. Step1-Step4 继承结论

Step5 继承以下前置结论：

- 公开门户只展示职位公开字段、企业公开字段、活动/内容公开字段和发布快照白名单字段。
- 求职者中心和企业中心都是私有域，运营后台不得把私有域数据复制到公开门户。
- `operator` 是运营写角色，`auditor` 是审计只读角色。
- 前端按钮、菜单和路由守卫只做体验；关键权限必须由服务端 RBAC、数据域和字段级权限强制。
- 审核、发布、下线、软删除、导出审批、风险处理等高风险动作必须写 `audit_log`。
- 查看敏感字段或高风险字段必须写 `field_access_log`。

## 3. Step5 总目标

Step5 的目标是把“运营如何维护站点、内容和风险”讲清楚。

| 能力域 | 目标 | 本文件冻结内容 |
| --- | --- | --- |
| 运营后台 | 让 operator 管理审核、内容、活动、风险和审计入口。 | 运营 overview、审核队列、内容管理、活动/招聘会管理、风险审核、审计入口。 |
| 内容活动 | 支撑公开门户的就业政策、资讯、HR 工具箱、公告和招聘会展示。 | CMS 字段白名单、软删除/下线、公开展示配置、SEO 口径。 |
| 权限审计 | 区分 operator 与 auditor。 | operator 可写，auditor 只读；所有写动作留痕。 |

Step5 不定义最终 API schema，不新增数据库表，不新增代码实现，只冻结运营后台页面范围、状态口径和验收边界。

## 4. 运营后台能力细化

### 4.1 运营首页 overview

运营首页应聚合站点运行和审核状态：

- 待审核企业数量。
- 待审核职位数量。
- 待审批导出数量。
- 待处理内容数量。
- 待处理招聘会/活动数量。
- 风险审核待处理数量。
- 最近 `audit_log` 摘要。
- 最近 `open_api_log` 异常摘要。

边界要求：

- overview 只展示运营摘要，不展示候选人手机号、邮箱、简历正文、附件对象 key 或证据。
- 统计口径必须可追溯，能通过 trace_id 或业务 ID 定位。
- auditor 可以查看只读 overview，不能触发处理动作。

### 4.2 企业审核

企业审核承接一期企业认证状态机：

- 待审核。
- 已通过。
- 已驳回。

运营后台应展示：

- 企业公开名称。
- 城市、行业、规模等公开摘要。
- 认证状态。
- 提交时间。
- 驳回原因。
- 审核人和审核时间。

禁止展示：

- 营业执照附件。
- 审核材料原文。
- 企业后台联系人手机号、邮箱。
- 内部风控备注明文进入公开门户。

审核通过或驳回必须写 `audit_log`，驳回必须保留原因。

### 4.3 职位审核

职位审核承接一期职位状态机：

- 待审核。
- 在线。
- 下线。
- 审核驳回。

运营后台应展示：

- 职位标题。
- 企业公开名称。
- 城市、职类、薪资展示。
- 审核状态。
- 公开状态。
- 提交时间。
- 驳回原因。

禁止：

- 未通过企业认证的职位上线。
- operator 绕过审核状态直接让职位公开。
- auditor 执行职位审核写操作。
- 将内部审核备注带入公开职位详情。

职位审核、上线、下线、驳回必须写 `audit_log`。

### 4.4 导出审批

导出审批继续继承一期 export_apply 链：

- 企业提交导出申请。
- operator 审批通过或驳回。
- 审批通过后异步生成。
- 企业按短期 MinIO 预签名链接下载。

运营后台应展示：

- 导出申请 ID。
- 申请企业。
- 业务类型。
- 审批状态。
- 生成状态。
- 下载发放状态。
- 申请原因。
- 驳回原因。

禁止：

- 未审批下载。
- 审批未通过时显示下载入口。
- 绕过字段级权限导出明文手机号或邮箱。
- auditor 审批导出。

审批、驳回、生成、下载链接发放必须能在审计中心按 trace_id 串联。

## 5. 内容与活动能力细化

### 5.1 内容管理

内容管理服务公开门户的就业政策、职场资讯、HR 工具箱、公告和帮助中心。

允许规划字段：

- content_id。
- content_type。
- title。
- summary。
- cover_url。
- city_code。
- published_at。
- status。
- public_body 或公开内容块。
- seo_title、seo_description、seo_keywords。

禁止字段：

- 候选人手机号、邮箱。
- 简历正文。
- 附件对象 key。
- 同意或撤回证据。
- 企业营业执照附件。
- 企业审核材料。
- 内部风控备注。

内容删除语义应为软删除或下线，不做物理删除。创建、更新、发布、下线、软删除必须写 `audit_log`。

### 5.2 首页推荐位与运营配置

首页和频道页可规划轻量运营配置：

- 推荐职位。
- 推荐企业。
- 推荐招聘会。
- 推荐资讯。
- HR 工具箱推荐。
- 公告栏。

边界要求：

- 推荐位只能引用已公开可见的职位、企业、活动或内容。
- 已下线、审核驳回、未公开或已撤回的对象不得出现在推荐位。
- 推荐位调整必须写 `audit_log`。
- 不在 Step5 规划付费置顶、会员权益或广告售卖。

### 5.3 活动与招聘会管理

活动/招聘会管理服务公开招聘会页面与活动详情。

首批允许规划：

- 现场招聘会。
- 网络招聘会占位。
- 校园招聘占位。
- 活动列表。
- 活动详情。
- 主办方。
- 时间、地点、城市。
- 公开简介。
- 活动状态。

后置能力：

- 活动报名。
- 参会名单。
- 展位预定。
- 展位售卖。
- 在线互动。
- 直播间。
- 真实地图定位。

活动创建、更新、发布、下线、软删除必须写 `audit_log`。

### 5.4 招聘会公开展示配置

招聘会公开展示必须遵守公开字段白名单：

允许展示：

- 活动 ID。
- 活动标题。
- 活动类型。
- 城市。
- 地点。
- 开始时间。
- 结束时间。
- 主办方。
- 公开简介。
- 状态。

禁止展示：

- 候选人报名名单。
- 候选人联系方式。
- 企业联系人手机号、邮箱。
- 签到证据。
- 内部审核备注。
- 非公开附件。

网络招聘会、校园招聘、视频招聘和直播招聘只允许作为活动类型或占位入口，不接真实在线互动、视频或直播服务。

## 6. 风险审核与风险池

### 6.1 风险审核对象

Step5 只冻结风险审核的运营视角，不实现自动风控或外部审查系统。

风险审核对象包括：

- 企业资料风险。
- 职位内容风险。
- 内容发布风险。
- 活动/招聘会风险。
- 导出申请风险。
- 审计异常风险。
- 对接接口异常风险。

风险审核必须区分：

- 当前可处理的低风险运营事项。
- 需要补充材料的事项。
- 进入风险池、暂不开发或暂不上线的高风险能力。

### 6.2 风险池规则

以下能力只能进入风险池，不自动进入开发：

- 受控找人才。
- 联系解锁。
- 会员商业化。
- 真实支付。
- 真实短信、微信、小程序、App。
- 真实地图、视频、直播。
- 真实 ATS/RPA 或受控候选池同步。
- AI/模型分析候选人原始数据。

风险池条目至少记录：

- 风险主题。
- 触发场景。
- 涉及数据。
- 当前结论。
- 未来进入条件。
- 需要的审计与测试。

## 7. 审计中心运营视角

运营后台应提供审计中心入口，但审计中心仍保持只读查询属性。

运营可按以下维度查询：

- trace_id。
- 操作人。
- 角色。
- 业务类型。
- 业务 ID。
- 动作类型。
- 时间范围。
- open_api client_code。

响应必须统一脱敏：

- 手机号。
- 邮箱。
- 简历正文。
- 附件对象 key。
- 证据。
- password。
- secret。
- token。

auditor 可以查看审计中心；operator 可以查看审计中心；candidate/company 不能进入运营审计中心。

## 8. 角色权限边界

| 角色 | 允许 | 禁止 |
| --- | --- | --- |
| operator | 审核企业、审核职位、审批导出、管理内容、管理活动、处理风险、查看审计。 | 绕过字段级权限、绕过导出审批、查看未授权原始候选人数据。 |
| auditor | 查看审核队列、查看内容活动、查看审计中心、查看风险池摘要。 | 调用任何写接口；审核、发布、下线、软删除、审批导出、调整推荐位。 |
| company | 查看本企业业务后台。 | 访问运营后台、审批导出、审核职位、查看其他企业数据。 |
| candidate | 查看本人求职者中心。 | 访问运营后台、查看企业审核材料、查看审计中心。 |

前端可根据角色隐藏按钮，但不得把隐藏按钮当作安全控制。后续实现必须用服务端权限拒绝 auditor 写操作，并返回统一错误与 trace_id。

## 9. 运营状态机

| 状态 | 适用对象 | 表现 |
| --- | --- | --- |
| loading | 审核队列、内容、活动、审计查询 | 加载中。 |
| empty | 队列无数据 | 展示暂无待处理。 |
| unauthorized | 未登录或 token 失效 | 提示登录。 |
| forbidden | 非运营角色或 auditor 写操作 | 展示无权限和 trace_id。 |
| pending_review | 企业、职位、内容、活动、导出 | 待审核或待审批。 |
| approved | 企业、职位、导出 | 已通过。 |
| rejected | 企业、职位、导出 | 已驳回，显示原因。 |
| published | 内容、活动、推荐位 | 已公开展示。 |
| offline | 内容、活动、职位 | 已下线，公开不可见。 |
| soft_deleted | 内容、活动 | 已软删除，后台可追溯。 |
| risk_pool | 高风险能力 | 暂不开发或暂不上线。 |

状态必须由服务端返回，前端不得用按钮显示、字段缺失或本地缓存推断审核结果。

## 10. 自动化验收关注点

后续实现 Step5 相关能力时，至少要覆盖：

- operator 可执行审核、发布、下线、软删除和导出审批。
- auditor 调用任何写接口返回 `AUTHZ_403` 并包含 trace_id。
- 内容和活动删除是软删除或下线，不物理删除。
- 公开内容、招聘会和活动不返回手机号、邮箱、简历正文、附件对象 key、证据或审核材料。
- 推荐位不得引用未审核、已下线、已驳回或已撤回对象。
- 招聘会公开展示不包含报名名单、联系方式、签到证据或内部备注。
- 审核、发布、下线、软删除、风险处理和导出审批写 `audit_log`。
- 敏感字段读取写 `field_access_log`。
- 审计中心响应脱敏，且可按 trace_id 串联。

## 11. Step5 结论

Step5 冻结了二期运营后台与内容活动能力边界：

- 运营后台聚焦审核、内容、活动、风险和审计入口。
- 内容活动聚焦就业政策、资讯、HR 工具箱、公告、首页推荐位和招聘会公开展示配置。
- operator 是授权写角色，auditor 是只读角色。
- 高风险能力进入风险池，不自动进入开发。
- 所有运营写动作必须审计，所有敏感读取必须可追溯。

高保真门户优先补充：

- Step5 的运营后台能力继续保留，但二期主线会先把公开门户做成接近 demo 的完整人才网，再增强运营后台。
- 内容、活动、招聘会和推荐位配置必须服务于首页高保真模块、招聘会频道、资讯/HR 工具箱频道。
- operator/auditor 权限边界不因前端门户视觉增强而放松。

下一步建议有两个方向：

1. 进入 `二期研发实施蓝图与任务拆解.md`，把 Step1-Step5 拆成可开发里程碑。
2. 若仍需继续细化风险能力，则先写 `LocalTalent_phase2_step6.md：风险池与高风险能力准入规则`。

本文件不生成 Step6、二期蓝图或二期提示词。
