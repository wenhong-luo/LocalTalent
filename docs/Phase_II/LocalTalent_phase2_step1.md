# LocalTalent Phase-2 Step 1：目标站资料读取与门户信息架构冻结

## 开发前必读卡

- **文件定位**：二期重做后的第一个 Step，只冻结产品定位、目标站结构、首批门户导航、页面域和隐私边界。
- **实现约束**：本文件是规划输入，不是开发 Prompt；不新增代码、不新增接口、不新增数据库迁移。
- **禁止越界点**：禁止把 demo 中的“简历库/搜索简历/找人才”照搬为公共简历库；禁止公开联系方式；禁止真实 ATS/RPA 同步；禁止真实受控候选池同步。
- **下游影响**：为 Step2 门户公开页面、Step3 转化闭环、Step4 双中心能力细化提供产品域边界。
- **自动化验收关注点**：后续实现必须继续守住人才服务区、发布快照、原始候选人数据、同意、撤回、对接接口、数据域、字段级权限。

**统一硬边界提醒**：人才服务区仅展示发布快照；对接接口仅 stub；不得输出原始候选人数据；同意、撤回、数据域、字段级权限必须在服务端收口。

## 3. Demo 目标站信息架构

### 3.1 首页

demo 首页包含城市切换、求职者/招聘者登录注册、职位/企业/人才/资讯搜索、扫码关注、职类导航、招聘会、就业政策、更多服务和管理中心入口。

LocalTalent 取舍：

- 保留首页、城市、搜索、核心频道入口。
- 保留求职者/企业双入口。
- 保留扫码/移动端作为展示占位，但二期 Step1 不规划真实微信、小程序、App 接入。

### 3.2 找工作与名企直聘

demo 找工作和名企直聘页面包含地区、职类、薪资、更新时间筛选，以及最新职位、紧急招聘、智能匹配、快速招聘等入口。

LocalTalent 取舍：

- 二期第一阶段应优先规划找工作列表、职位详情、名企/认证企业推荐。
- 紧急招聘、今日招聘、快速招聘先作为运营标签或占位。
- 智能匹配需要画像和解释性，后续评估，不作为首批能力。

### 3.3 找企业

demo 找企业页面包含地区、企业性质、行业类别、企业列表和地图入口。

LocalTalent 取舍：

- 二期第一阶段应规划企业公开列表和企业主页。
- 地图入口只做后续占位，不接真实地图服务。
- 企业公开页不得展示联系人手机号、邮箱、营业执照附件、审核材料。

### 3.4 简历库

demo 简历库页面包含地区筛选、职类筛选、简历列表、搜索简历/找人才入口。

LocalTalent 取舍：

- 不照搬公共简历库。
- 游客公开侧统一改为人才服务区，只展示发布快照摘要。
- 企业侧未来可规划受控找人才，但必须登录、企业认证通过、服务端权限通过，并按字段级权限裁剪。
- 联系方式默认不可见。

### 3.5 招聘会、网络招聘会、校园招聘

demo 招聘会相关页面包含现场招聘会、网络招聘会、校招职位、院校、双选会、宣讲会等。

LocalTalent 取舍：

- 二期第一阶段可规划招聘会/活动公开列表与详情。
- 在线预定、报名、展位费用、参会名单先作为后续 Step 拆分。
- 校园招聘可以作为“更多服务”的后续频道，不进入 Step1 首批实现。

### 3.6 更多服务

demo 更多服务包含今日招聘、地图找工作、快速招聘、校园招聘、求职登记、网络招聘会、视频招聘、直播招聘、自由职业、帮助中心。

LocalTalent 取舍：

- 今日招聘、帮助中心可作为轻量信息入口规划。
- 地图找工作、视频招聘、直播招聘、自由职业先只做占位，不接外部服务。
- 快速招聘、求职登记涉及联系方式和个人信息收集，暂不进入首批实现。

### 3.7 HR 工具箱与就业政策

demo HR 工具箱包含招聘/面试、人事档案、员工管理、合同/试用、薪酬制度、加班/考勤、出差、培训、绩效考核等资料分类。

demo 就业政策/资讯包含就业人才、职业指导、简历指南、面试宝典、职场八卦、劳动法苑、职场观察等分类。

LocalTalent 取舍：

- 二期可以把 HR 工具箱、就业政策、职场资讯合并为“内容中心”规划。
- 公开内容只展示 CMS 公开字段，不涉及候选人原始数据。

### 3.8 登录注册与管理中心

demo 登录注册包含求职者会员登录/注册、招聘者会员登录/注册、手机验证码登录、微信扫码登录、忘记密码、用户协议与隐私政策、滑块验证。

LocalTalent 取舍：

- LocalTalent 一期已有本地账号 + JWT。
- 二期可规划正式登录页和注册页，但真实短信、微信扫码、小程序、App 登录先不接。
- 管理中心继续分为求职者中心、企业中心、运营后台，不混用权限。

## 5. 门户导航规划

二期首批门户导航冻结为：

```text
首页 / 找工作 / 找企业 / 人才服务区 / 招聘会 / 就业政策 / 更多服务 / 管理中心
```

导航口径：

- **首页**：聚合搜索、城市、热门职位、推荐企业、招聘会、资讯入口。
- **找工作**：公开职位列表和职位详情。
- **找企业**：公开企业列表和企业主页。
- **人才服务区**：公开发布快照摘要；不是公共简历库。
- **招聘会**：现场招聘会/活动列表与详情。
- **就业政策**：就业政策、职场资讯、HR 工具箱内容入口。
- **更多服务**：今日招聘、网络招聘会、校园招聘、求职登记、地图找工作、视频招聘、直播招聘、自由职业等占位入口。
- **管理中心**：根据身份进入求职者中心、企业中心或运营后台。

需要特别改写的 demo 导航：

- demo 的“简历库”公开侧改名为“人才服务区”。
- demo 的“搜索简历/找人才”企业侧另规划为“受控找人才”，不在游客公开导航直接暴露高敏简历数据。

## 6. 页面域规划

### 6.1 公开门户域

面向游客和搜索引擎，允许展示：

- 职位公开信息。
- 企业公开信息。
- 招聘会/活动。
- 就业政策/资讯/HR 工具箱。
- 发布快照白名单摘要。

禁止展示手机号、邮箱、简历正文、附件对象 key、证据、聊天记录、审核材料、营业执照附件。

### 6.2 求职者中心域

面向 candidate 本人，允许管理：

- 本人简历。
- 本人投递。
- 本人面试。
- 本人收藏/订阅。
- 本人同意/撤回。
- 本人通知。

必须服务端校验本人数据域。

### 6.3 企业中心域

面向 company 用户，允许管理：

- 本企业资料。
- 本企业职位。
- 本企业投递池。
- 面试邀约。
- 导出申请。
- 未来受控找人才。

必须服务端校验企业数据域；未认证企业不得使用高风险人才检索能力。

### 6.4 运营后台域

面向 operator 和 auditor：

- operator 可审核和配置。
- auditor 只读。
- 所有审核、上下线、推荐位、内容发布、风险处理必须写 `audit_log`。

### 6.5 开放接口 Stub 域

继续保持：

- 对接接口仅 stub。
- 不做真实 ATS/RPA 同步。
- 不做真实受控候选池同步。
- 不传输原始候选人数据。

## 7. Step1 结论

Step1 已完成目标站资料读取、一期基线回顾、首批门户导航、页面域和隐私边界冻结。

当前补充结论：

- 二期不再只是“频道功能清单”，而是以 `docs/Phase_II/demo` 的 24 个 HTML 页面为目标效果来源。
- `LocalTalent_phase2_demo_inventory.md/html` 已成为 Step1 与后续 Prompt 的桥接资料，负责把 demo 首页、列表页、登录注册和更多服务映射到 Prompt 15-23。
- 二期第一开发阶段调整为 **高保真门户优先**：先做公共门户 Shell、首页首屏、找工作、找企业、人才服务区改写、招聘会/资讯/HR 工具箱，再做更深的中心增强。
- Step1 的隐私边界保持不变：demo 的“简历库/搜索简历/找人才”只能改写为人才服务区或未来受控找人才，不得成为公共简历库。

已完成的后续文档包括 Step2-Step6、二期蓝图和二期提示词。后续开发应从 `二期研发提示词.md` 的 Prompt 15（P2-0）开始，不得跳过 demo 同步与页面盘点闸门。
