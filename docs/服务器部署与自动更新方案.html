LocalTalent 服务器部署与自动更新方案

VPS 源码部署、服务器构建、systemd 重启，以及 push 后自动更新的落地说明。

结论 推荐架构 服务器准备 首次部署 手动更新 自动更新 部署验收 安全边界

1. 结论先行

  • 可以部署到服务器上。
  • 本地 git commit 不会自动影响服务器；必须至少 git push 到远程仓库。
  • 服务器要更新，需要执行 git pull 或由 webhook / CI 触发部署脚本。
  • 前端是 Next.js SSR，生产运行依赖 .next 构建产物，所以正式部署必须重新 npm run build。
  • 后端 Java 代码变化后也必须重新 ./mvnw clean package 并重启服务。
不建议用 npm run dev 给领导或灰度环境看；开发模式不代表试运营形态。

4. 首次部署

sudo -iu localtalent
git clone https://github.com/wenhong-luo/LocalTalent.git /opt/localtalent/app
cd /opt/localtalent/app
docker compose -f infra/docker-compose.yml up -d
sudo cp .env.phase3.example /etc/localtalent/backend.env
sudo cp .env.phase3.example /etc/localtalent/frontend.env
sudo chmod 600 /etc/localtalent/*.env
./scripts/check_boundary
./scripts/check_portal_snapshot_fields
./scripts/check_phase3_boundary
./scripts/check_phase3_ops
./scripts/check_phase3_security_acceptance
./scripts/check_phase3_gray_acceptance

cd backend
./mvnw clean package

cd ../frontend
npm ci
npm run build
sudo cp deploy/systemd/localtalent-backend.service.example /etc/systemd/system/localtalent-backend.service
sudo cp deploy/systemd/localtalent-frontend.service.example /etc/systemd/system/localtalent-frontend.service
sudo systemctl daemon-reload
sudo systemctl enable --now localtalent-backend localtalent-frontend

5. 手动更新流程

cd /opt/localtalent/app
git pull --ff-only

./scripts/check_boundary
./scripts/check_portal_snapshot_fields
./scripts/check_phase3_boundary
./scripts/check_phase3_ops
./scripts/check_phase3_security_acceptance
./scripts/check_phase3_gray_acceptance

cd backend && ./mvnw clean package
cd ../frontend && npm ci && npm run build

sudo systemctl restart localtalent-backend
sudo systemctl restart localtalent-frontend

6. 推荐自动更新：push 后部署

本地 commit -> git push origin main -> 服务器 webhook / CI 触发 -> git pull -> 检查 -> 构建 -> 重启服务

服务器上可执行：

cd /opt/localtalent/app
LOCALTALENT_DEPLOY_CONFIRM=deploy ./scripts/deploy_server_source

如果只演练，不重启服务：

LOCALTALENT_DEPLOY_CONFIRM=deploy \
LOCALTALENT_DEPLOY_RESTART_MODE=none \
./scripts/deploy_server_source
不要在 webhook 中执行未校验来源的 shell 命令；自动部署必须校验 webhook secret 或由可信 CI 触发。

8. 安全边界

  • 人才服务区仍只展示发布快照。
  • 公开门户不输出原始候选人数据。
  • 对接接口仍为 stub。
  • 公共简历库、受控找人才、联系解锁、真实支付、真实短信/微信/小程序/App、真实地图/视频/直播、ATS/RPA 仍不得上线。
  • 生产/灰度环境不要使用模板中的占位密钥。
